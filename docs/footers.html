Footers

Four footer variants, sized to the weight of the site: compact for tools, standard for marketing, expanded with a CTA, and inverse for high-emphasis closes.

Anatomy

A footer holds a brand block, optional link columns, an optional CTA row, and a bottom row with copyright and legal links. The compact variant collapses everything into a single row.

Compact
R Riverside

© 2026 Riverside Medical Center. All rights reserved.

When to use

  • Use compact on tools, dashboards, and internal docs where the footer is a sign-off, not a sitemap.
  • Use standard (default) on marketing and content sites that need clear navigation by topic.
  • Use expanded when a final conversion moment belongs at the bottom of the page (appointment, donation, signup).
  • Use inverse when the page above has finished light and a dark close gives a strong, intentional bookend.

When not to use

  • Do not duplicate the top nav in the footer; the footer is for breadth (sitemap), not active wayfinding.
  • Do not place a primary CTA in a compact footer; reach for expanded.
  • Do not use inverse if the section above is already dark — the page loses its rhythm.

Accessibility

  • Footer must use the <footer> landmark element so screen readers announce it as a region.
  • Each link list belongs inside a <ul role="list">; the column title above is a heading-style label, not a true heading, so it stays a <div>.
  • Inverse variant tweaks link colors to keep 4.5:1 contrast against the dark surface.
  • Avoid hiding legal links behind tiny font sizes — the system uses 12px minimum and clear underline-on-hover.
