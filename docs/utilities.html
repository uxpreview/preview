Utilities

Single-purpose helper classes for the cases a layout primitive doesn't cover — text alignment and color, backgrounds, visibility, and one-off width and spacing. Reach for these last, after the layout primitives.

Width & spacing helpers

Tiny escape hatches. Prefer stack/cluster gap for spacing; reach for margin utilities only when no parent gap fits.

.u-full-width / .u-max-prose

A paragraph constrained to roughly 65 characters per line — useful for placing text inside containers wider than comfortable reading.

When to reach for utilities

  • Use a utility when you need a single, one-off behavior that does not belong in a component.
  • Prefer layout primitives (.u-stack, .u-cluster, .u-grid) over margin utilities; gap-based spacing scales better.
  • Use .u-visually-hidden liberally for screen-reader labels — it is the cheapest accessibility win in the system.

When not to use a utility

  • Do not build a recurring pattern from a stack of utilities; once you reach for three or more, the pattern wants to be a component.
  • Do not chain margin utilities to recreate a stack — use the stack primitive.
