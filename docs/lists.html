Lists

Three list patterns for three jobs: a large clickable row, a label-and-value pairing, and a news-style article list with meta column.

Anatomy

A linked list is an <ul> of rows where the whole row is a link. A definition list pairs terms with descriptions. An article list arranges editorial entries with a meta column and a content column.

Linked list — default
  • Spinal cord injury14 conditions
  • Stroke22 conditions
  • Brain injury18 conditions

When to use

  • Use a linked list for category indexes — long lists where every entry is a destination.
  • Use a definition list for labelled facts (insurance accepted, visit types, credentials).
  • Use an article list for news, research, or story indexes where each item has a date and category.

When not to use

  • Do not use a linked list when items need images or rich previews — reach for cards.
  • Do not use a definition list as a workaround for two-column layout; use .u-grid or .wire-twocol.
  • Do not nest article lists; one flat sequence per page section.

Accessibility

  • Use <ul role="list"> so Safari preserves list semantics when list-style is removed in CSS.
  • Each linked list row is a single anchor with the full text — do not nest additional interactive elements inside the row.
  • Definition lists must use <dl>, <dt>, and <dd>; the term-description pairing carries semantics no styling can replace.
  • Article list items use <article>; the date uses <time> with a datetime attribute in production.

    </a>
  </li>
</ul>
