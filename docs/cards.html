Component

Cards

A single shell — .wire-card — extended by purpose: content, person, resource, stat. Variants change emphasis (subtle, bordered, inverse, linked), never identity.

Anatomy

Every card is a container, an optional media slot, and a body with eyebrow, title, description, optional meta row, and an optional footer that pins to the bottom on equal-height grids.

When to use

  • Use a content card for a teaser to a long-form item (article, story, news).
  • Use a person card for staff, clinicians, researchers — anything with a face and a role.
  • Use a resource card for a downloadable or watchable asset; the resource type sits in the eyebrow slot.
  • Use a stat card to land a single number prominently in a grid.

When not to use

  • Do not use a card when a linked list will do — long single-line entries belong in .wire-linked-list.
  • Do not mix card types in the same grid; the silhouette break is jarring.
  • Do not wrap a card in a stray <a>; link the title and use .wire-card--linked for the hover affordance.

Variants

Purpose modifiers come first (person, resource, stat); style modifiers stack on top (subtle, bordered, inverse, linked).

Style — subtle, bordered, inverse, linked
Subtle

Quieter surface

Use inside a white section to pull the card back without losing its container.

Bordered

Stronger frame

A heavier border when the card needs to feel separated from a busy background.

Inverse

Dark moment

For high-emphasis CTAs or featured editorial blocks inside a light page.

Linked

Whole card is clickable

Hover gives a strong border to signal the entire card is the affordance.

Accessibility

  • Wrap content cards in <article>; stat and resource cards in <div> unless the content is article-like.
  • Link the title only; do not link the whole card via nested anchors. Use .wire-card--linked for the hover affordance and rely on the title link for navigation.
  • Person card images are decorative greyboxes here; in production, ensure the alt text describes the person if the photo is meaningful, or leave it empty if the name beside it carries the meaning.
  • Maintain a consistent eyebrow→title→description→meta order so screen readers traverse a predictable structure.
