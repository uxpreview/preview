v1.4 · Feedback & communication

Tell the user what's happening, then stay out of the way.

Seven patterns for the moments when the system needs to say something — confirm a destructive action, acknowledge a save, warn about an outage, show a loading state, or explain an empty list. Each pattern is the lightest tool that does the job.

Pick the lightest pattern

From least to most intrusive. The user's tolerance for interruption shrinks fast — reach for a heavier pattern only when the message earns it.

Pattern Use when Avoid when
BannerSystem-level context the user should know about while still using the page (downtime, policy, outage).The message is transient or about one action they just took.
ToastYou just acted on the user's behalf and want to confirm it (saved, sent, deleted).The user needs to acknowledge the message before continuing — use a modal.
TooltipA control's label is brief and a sentence of context would help. Hover and focus reveal.The content is interactive, long, or critical — interactive content fails WCAG 1.4.13.
SkeletonYou know what's loading and the layout, but the data isn't here yet.The wait is unbounded or unknown — show a spinner or a progress bar.
Empty stateA list is empty by design or because of a filter — explain and offer a path forward.The state is loading (use skeleton) or errored (use a callout).
StepperThe flow has 3–7 discrete steps and the user benefits from seeing progress.The flow is one step (no progress to show) or branchy (the linear visual lies).
ModalYou need the user to confirm or commit before continuing. Blocking.The information is just informative — a banner or a callout works without trapping focus.

wire-banner

Full-width announcement strip. Sits at the top of the page or above a section. Reserved for context that affects the whole experience, not transient feedback.

i
Updated visitor policy
As of May 26, masks are no longer required in waiting areas. Inpatient unit policies have not changed.
Read the changes
!
Patient portal scheduled maintenance — Sunday 2–4 AM CT
Appointment booking will be temporarily unavailable. Call 312.555.1000 if you need urgent assistance.

Accessibility

  • For high-urgency content (outages), wrap the banner in <div role="alert"> so AT users hear it immediately. Otherwise the banner is just text.
  • The icon is decorative — keep aria-hidden="true".
  • The dismiss button is removed by js/wire.js; once dismissed the user can't recover it. Don't put critical info in a dismissable banner.

wire-toast

Slide-in confirmation for an action the user just took. Auto-dismisses in 5 seconds by default. Rendered into a single polite live region per page so screen readers hear the message without re-announcing on every visual update.

Click the buttons to fire a toast

JS API

The script exposes window.wireToast({ title, body, duration }). Calling it from any inline handler creates a toast. Duration of 0 disables auto-dismiss (the user must close it).

Accessibility

  • The region uses role="status" + aria-live="polite" per WCAG 4.1.3.
  • Don't use toasts for errors the user must address — errors belong inline next to the field, or in a banner / modal.
  • Don't queue more than 3 toasts. If you're firing many in a row, batch them or use a banner.

data-wire-tooltip

Pure-CSS hover/focus tooltip. Attach data-wire-tooltip="text" to any element. The text appears on hover and on keyboard focus.

Hover or tab to each control
Last saved 2 minutes ago

Accessibility (WCAG 1.4.13)

  • Tooltips appear on focus, not just hover — keyboard users see them too.
  • Content is plain text only. For interactive content (links, buttons inside), use a popover or render the help inline.
  • For icon-only buttons, the tooltip is supplementary — always pair with aria-label as the accessible name.

wire-skeleton

Animated placeholder for content that is still loading. Reserves the layout so when real content arrives there's no jump.

Card skeleton

Accessibility

  • Wrap a skeleton block in a container with aria-busy="true" and aria-live="polite"; AT users hear the change once real content swaps in.
  • The shimmer respects prefers-reduced-motion: reduce — animation stops, the gray block stays visible.

wire-empty

Three jobs: explain why the list is empty, reduce confusion, offer a path forward.

∅

No upcoming visits

You don't have any appointments scheduled. Find a doctor to book one, or call us at 312.555.1000.

Find a doctor Browse conditions

When not to use

  • When the list is loading — use wire-skeleton.
  • When there's an error — use wire-callout or wire-banner with a specific recovery action.

wire-stepper

Progress indicator for a multi-step flow. Horizontal at tablet+, vertical on phones.

  1. Choose provider Dr. Okonkwo
  2. Pick a time Tue, June 9, 10:30 AM
  3. Confirm Review & submit
  4. Done

Accessibility

  • The list uses <ol> — order conveys progression.
  • Set aria-current="step" on the current item.
  • Visual marker (✓ on complete, number on incomplete) is generated by CSS counter — don't rely on it to convey state, the is-complete/is-current class is the source of truth.

wire-modal

Blocking dialog. Use when the user must confirm or commit before continuing. Implements the ARIA dialog pattern: focus trap, Esc closes, backdrop click closes, focus returns to the trigger.

Cancel this appointment?

Cancelling will release Tuesday's 10:30 AM slot. You'll need to start over to rebook.

Accessibility (ARIA APG dialog pattern)

  • role="dialog" + aria-modal="true" set by JS.
  • The trigger gains aria-expanded + aria-controls.
  • Focus moves to the first focusable element on open, traps until close, returns to the trigger on close.
  • Esc closes. Backdrop click closes (don't use a modal as a confirmation if backdrop dismissal would be confusing — for now, all our modals allow it).

See it in real pages

  • hospital-appointment.html — a confirmation flow that exercises banner, stepper, tooltip, modal, toast, skeleton, and empty state in one realistic layout.
