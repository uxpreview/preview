Component

Media

Greybox placeholders for image and video, plus a small gallery system. Used everywhere a real asset will later live — and intentionally honest about its absence.

Anatomy

A single .wire-media element with a ratio modifier (or default 16:9). Add a data-ratio attribute to print a small ratio label in the top corner. Add a .wire-media__label child for a centered caption.

Default (16:9)

When to use

  • Use a placeholder for every asset slot at wireframe stage — including thumbnails, hero images, and headshots.
  • Use the ratio that matches the eventual asset (16:9 for video stills, 1:1 for headshots, 4:3 for editorial photos).
  • Use --video for any moving-image slot; the play triangle is recognisable as "this will be a video."
  • Use a gallery when three or more placeholders sit together; otherwise place a single .wire-media.

When not to use

  • Do not use a media placeholder when the page is content-only — leave the area blank rather than implying an asset will arrive.
  • Do not stretch a ratio modifier with inline styles; pick the closest preset.
  • Do not wrap a placeholder in a link without an accessible name.

Accessibility

  • Wireframe placeholders are decorative; real assets must replace them with <img> + meaningful alt, or empty alt for purely decorative images.
  • Video placeholders must become <video> with captions, transcripts, and audio descriptions where applicable.
  • Galleries should be wrapped in a <figure> or list when they form a meaningful set; otherwise plain divs are fine.
  • The data-ratio label is visual only — it provides no semantics and is safe to omit in production.

Show snippet 
<div class="wire-media wire-media--16-9" data-ratio="16:9">
  <span class="wire-media__label">Hero image — patient on therapy treadmill</span>
</div>

  <div class="wire-media wire-media--square" data-ratio="1:1"></div>
  <div class="wire-media wire-media--square" data-ratio="1:1"></div>
  <div class="wire-media wire-media--square" data-ratio="1:1"></div>
</div>
