Tables

A workhorse data table and a comparison table for plans, programs, and feature matrices. Both scroll horizontally on small screens via the required .wire-table-wrap container.

Anatomy

Every table is wrapped in .wire-table-wrap for overflow handling. The table itself uses standard HTML semantics — thead, tbody, th[scope] — with optional numeric column alignment via .wire-table__num.

When to use

  • Use a data table when rows share a structure and users need to scan across columns.
  • Use striped when row count crosses ten and zebra striping helps the eye stay on row.
  • Use compact in dashboards or admin contexts where vertical density matters.
  • Use a comparison table when three or fewer options are being weighed against the same criteria.

When not to use

  • Do not use a table for layout; use .u-grid or .u-sidebar instead.
  • Do not use a table for two- or three-row label-value content; use .wire-deflist.
  • Do not nest tables; flatten the data or split into multiple tables.

Accessibility

  • Always set scope="col" on header cells and scope="row" on row headers — screen readers use these to announce context.
  • Provide a <caption> for every data table; use .u-visually-hidden if a visible caption would be redundant.
  • Use .wire-table__num for numeric columns so digits align and use tabular-nums for stable column widths.
  • The comparison .wire-table__check and .wire-table__cross include text marks in CSS (✓ / —); add visually-hidden text inside the span if assistive tech needs literal "yes"/"no".
  • Horizontal scrolling via .wire-table-wrap keeps small screens usable; never collapse a real data table into cards on mobile.
