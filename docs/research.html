Why we made the calls we made.

How to read this

Every claim below carries one of two tags.

Evidence backed by a published standard or peer-reviewed/industry research
Convention aesthetic or designer judgment that follows industry practice but isn't independently validated

Where two reasonable options exist and we picked one, we say so. Where a choice is genuinely backed by data, we cite the source.

WCAG 2.2 AA compliance map

The system targets WCAG 2.2 Level AA. The table below maps each criterion the system actively meets to the implementation, and flags exceptions.

Criterion What it requires How we meet it
1.3.1 Info and Relationships Information conveyed through presentation can be programmatically determined. Semantic HTML throughout: nav, main, article, section, headings ordered, lists marked up as lists, form labels associated with inputs.
1.4.3 Contrast (Minimum) 4.5:1 for normal text, 3:1 for large. Grayscale ramp designed against this: gray-50 (#767676) on white = 4.6:1; default text gray-95 = 19:1. See ramp in tokens.css.
1.4.10 Reflow Content reflows to 320 CSS px wide without horizontal scrolling. Mobile-first layouts using grid-template-columns: 1fr defaults and auto-fit, minmax(min(var, 100%), 1fr) on grids. No fixed-pixel min-widths above 320px.
1.4.11 Non-text Contrast 3:1 for UI components and graphical elements. --color-border-strong (gray-50 = 4.5:1) used for form input borders, secondary buttons, card-bordered. Bumped from gray-40 in v1.1 audit pass.
1.4.12 Text Spacing No loss of content when line height ≥1.5, paragraph spacing ≥2× font size, etc. Line height 1.5 default; no fixed text container heights; overflow-wrap: break-word on flow text.
2.1.1 Keyboard All functionality available from keyboard. Buttons and links are native; tabs implement arrow + Home/End; accordion uses native details; mega menu and drawer respond to Esc.
2.1.2 No Keyboard Trap Focus can move into and out of any component. Drawer focus trap is intentional and matches the dialog pattern: Esc and close-button release focus to the trigger. Implemented in v1.1.
2.3.3 Animation from Interactions Allow users to disable non-essential motion. @media (prefers-reduced-motion: reduce) zeroes --duration-* tokens at the root, killing transitions globally.
2.4.3 Focus Order Focusable components receive focus in an order that preserves meaning. Source order matches visual order on every layout primitive. No tabindex values > 0.
2.4.4 Link Purpose (In Context) The purpose of each link is determinable from text alone or with surrounding context. Linked-list, article-list, and breadcrumb patterns avoid "click here" / "read more" with no context. Icon-only buttons require aria-label.
2.4.6 Headings and Labels Headings and labels describe topic or purpose. Every page leads with one descriptive h1; sub-sections use h2/h3 in order. Form labels are required, not placeholder-only.
2.4.7 Focus Visible Keyboard focus indicator is visible. :focus-visible applies a 2px solid --color-focus ring with 2px offset on every interactive element. Inverse contexts override --color-focus to white. v1.1 fix.
2.4.11 Focus Not Obscured (Min) Focused element is not entirely hidden by author-created content. scroll-margin-block-start on h1–h6 leaves clearance for sticky chrome on anchor jumps. v1.1 fix.
2.5.8 Target Size (Minimum) Pointer targets at least 24×24 CSS px (with exceptions). Buttons and form controls ≥44px (also meeting Apple HIG and WCAG AAA 2.5.5). Footer, breadcrumb, and in-page-nav links bumped to 28px+ in v1.1.
3.1.1 Language of Page Default human language is programmatically determined. <html lang="en"> on every page.
3.3.1 Error Identification Errors are described in text. .wire-field__error renders next to the field with an associated aria-describedby.
3.3.2 Labels or Instructions Labels are provided when input is required. Every input requires a <label> via the .wire-field pattern. Placeholders never substitute for labels.
4.1.2 Name, Role, Value UI components expose name, role, value programmatically. Native elements where possible. ARIA on tabs (role="tab", aria-controls, aria-selected), mega menu (aria-haspopup, aria-expanded), breadcrumb (aria-current="page"), pagination (aria-current, aria-disabled), drawer (role="dialog", aria-modal="true").
4.1.3 Status Messages Status changes can be programmatically determined. Loading buttons accept aria-busy="true"; error messages live in associated regions. Inline form errors do not require additional role="alert" for static state.

Source: W3C Web Content Accessibility Guidelines (WCAG) 2.2, recommendation Oct 2023. w3.org/TR/WCAG22

Interaction patterns

Tabs — arrow keys move between tabs, Home/End jump to ends, Enter/Space activates Evidence W3C WAI-ARIA Authoring Practices Guide (APG): Tabs Pattern
Accordion — single-open mode, +/− affordance, native <details> for built-in semantics Evidence W3C ARIA APG Accordion Pattern; MDN guidance on native details
Mobile drawer — focus trap, role="dialog", aria-modal, Esc to close, focus restored to trigger Evidence W3C ARIA APG Modal Dialog Pattern

Pill button radius Convention Aesthetic carry-over from MotherDuck reference. No evidence pills outperform rounded rectangles for usability.

Visible focus ring with 2px offset Evidence WCAG 2.2 Success Criterion 2.4.13 (Focus Appearance) — the offset prevents the ring being lost against component borders.

Typography & reading

ChoiceTagBasis
Body text 16px (1rem) default, never below 14px in flow content Evidence NN/g: "Legibility, Readability, and Comprehension"; WebAIM guidelines; iOS HIG default body
Line height 1.5 for body, 1.1–1.25 for headings Evidence WCAG 1.4.12 (Text Spacing) sets 1.5 as the safe-resize baseline. Bringhurst, The Elements of Typographic Style, recommends tighter leading for display.
Max line length 65–70 characters (max-inline-size: 70ch) Evidence Bringhurst (45–75 char range); NN/g "Line Length: How Many Characters Per Line?" (Pernice 2024) — comprehension optimal at 50–75
1.25 modular type scale Convention Common practice (Material, Bootstrap, IBM Carbon). No published evidence of measurable readability difference vs. 1.2 or major third.
System sans default; serif used sparingly for emphasis (display, pull quotes) Convention NN/g: "Serif vs. Sans Serif Fonts: Is One Easier to Read?" finds no meaningful readability difference at 16px+. We use serif for editorial moments.
text-wrap: balance on headings Evidence W3C CSS Text Module Level 4. Adam Argyle (Chrome team): "CSS text-wrap: balance" — measurably reduces orphan-line awkwardness in headlines

Component basis

Every visual component in the system carries either an Evidence tag (a published standard or peer-reviewed source backs the decision) or a Convention tag (the choice follows widely-accepted industry practice but isn't independently validated). Components covered elsewhere on this page (forms, typography, tabs, accordion, drawer, mega menu, breadcrumb, pagination, in-page nav, hero, help bar, text-size, print) are listed once here for completeness.

Component Tag Basis
wire-badge Convention Industry consensus on inline status pills. References: Material Design 3 Badge, IBM Carbon Tag.
wire-callout Evidence NN/g "Visual Hierarchy in UX" on aside emphasis; WCAG 1.3.1 requires semantic asides be programmatically distinguishable.
wire-card Convention Card pattern is industry-standard for grouping related content. References: Material Design 3 Cards, IBM Carbon Tile.
wire-citation-list Convention Numbered citation entries with stacked author / title / source metadata follow APA 7th edition and Chicago Manual of Style visual conventions for reference lists.
wire-divider Convention Bringhurst's Elements of Typographic Style on visual rhythm — dividers as silent punctuation between sections.
wire-feature-grid Evidence NN/g "Cards: UI-Component Definition" on icon + title + description grids for landing pages.
wire-gallery Convention Wireframe industry standard for image-grid layouts; mosaic ratios chosen for visual interest, single-column collapse on mobile per WCAG 1.4.10.
wire-help-bar Evidence NIA "Making Your Website Senior Friendly" on persistent, high-contrast help affordances for older-adult users; Pew "Health Online 2013" on phone as a primary care-contact channel.
wire-linked-list Evidence NN/g "Category Pages Are Powerful" — large clickable rows with descriptive secondary text outperform compact lists on category landings.
wire-media Convention Grayboxed placeholders with ratio labels are the wireframe-industry convention for asset-deferred mockups. Marked aria-hidden="true" per WCAG 1.1.1.
wire-quote Evidence Bringhurst on pull quotes as visual rhythm; NN/g "Pull Quotes" on scannability and emphasis.
wire-sidenav (incl. local / secondary / section-tree compositions) Evidence NN/g "Left-Side Vertical Navigation" on left-rail orientation in deep IAs; IBM Carbon UI Shell Side Nav spec for hierarchy patterns. aria-current="page" per WCAG 4.1.2.
wire-table Evidence WCAG 1.3.1 (header scope); W3C ARIA APG Table pattern; Bringhurst on tabular alignment and tnum figures.
wire-tag Convention Chip / removable-tag pattern from Material Design 3 Chips and IBM Carbon Tag. wire-tag--sm meets WCAG 2.5.8 AA floor (24px).
wire-timeline Convention Vertical chronology with anchored date markers is consensus design practice. NN/g "Timelines in UX" on event-list scannability.
wire-twocol (and u-sidebar) Convention Two-column text + media and sidebar + main layouts trace to the Swiss-grid tradition; Bringhurst on column hierarchies and reading-line length (~66 characters).
wire-modal Evidence W3C ARIA APG Dialog (Modal) pattern for role / aria-modal / focus trap / Esc; NN/g "Modal vs. Nonmodal Dialogs" on when blocking is justified.
wire-toast Evidence WCAG 4.1.3 Status Messages — polite live region announces state changes without interrupting. NN/g "Indicators, Validations, and Notifications" on when to use transient vs. persistent feedback.
wire-banner Evidence NN/g "User Notifications: UX Guidelines" on site-level vs. transient messages; WCAG 1.3.1 requires the alert's region be programmatically distinguishable. The dismissable variant follows WCAG 2.2.2 (Pause, Stop, Hide).
wire-empty Evidence NN/g "Empty State Interface Design" — three jobs: explain, reduce confusion, offer recovery. Bordered variant from IBM Carbon Empty pattern.
wire-skeleton Evidence NN/g "Progress Indicators" on reducing perceived wait time. Shimmer respects prefers-reduced-motion per WCAG 2.3.3. Pairs with aria-busy / aria-live.
wire-stepper Convention Step indicators are an established multi-step-flow pattern. Reference: Material Design 3 Steppers, IBM Carbon Progress Indicator. ARIA: <ol> + aria-current="step".
data-wire-tooltip Evidence WCAG 1.4.13 Content on Hover or Focus — dismissable, hoverable, persistent. ARIA APG Tooltip pattern guidance; for icon-only buttons, paired with aria-label as the accessible name (the tooltip is supplementary).

Patterns added or emphasized for healthcare engagements (Riverside and similar).

What's convention, not evidence

The system makes plenty of choices that are aesthetic or consensus-driven. We don't dress them up as research.

  • Grayscale ramp granularity (12 steps). Designer judgment to provide enough hierarchy without overwhelming the file. Material uses 10, IBM Carbon uses 9.
  • 4px spacing base. Industry consensus (Material, Carbon, Polaris). 8px base would also be defensible.
  • Pill button radius. Aesthetic choice from the MotherDuck reference. No evidence pills outperform rounded rectangles.
  • Modular 1.25 type scale. Common; no evidence of measurable advantage over 1.2 or major third.
  • Specific motion durations (150/250/400ms). Material guideline-aligned but no peer-reviewed support for these exact numbers.
  • System-font default over a custom face. Portability decision — every consumption surface (browser, Figma Make, artifacts) gets a faithful rendering without a network dependency.

Audit history

Two documented WCAG 2.2 AA passes have run against the system. Each pass produced a table of findings and the specific fix; both are reproduced below for the public record.

v1.1 — initial audit, found and fixed

CriterionFindingFix
1.4.11 Non-text Contrast --color-border-strong was gray-40 = 2.4:1 vs white. Form input borders failed the 3:1 UI minimum. Bumped to gray-50 = 4.5:1. Cascades to forms, secondary buttons, tags, toggles, accordion icon, card-bordered.
2.4.7 Focus Visible --color-focus was gray-95, identical to --color-bg-inverse. Focus ring invisible on dark heroes, footers, callouts. Component-scoped override: .u-bg-inverse, .wire-hero--inverse, .wire-footer--inverse, .wire-callout--inverse set --color-focus: var(--color-text-inverse).
2.1.2 No Keyboard Trap Mobile drawer lacked focus trap. Tab moved focus to elements behind backdrop while drawer was open. Drawer now implements modal dialog pattern: role="dialog", aria-modal="true", focus trap on Tab/Shift+Tab, Esc closes, focus returns to trigger.
2.4.11 Focus Not Obscured Anchor jumps to in-page sections could land target under sticky chrome on small viewports. scroll-margin-block-start: var(--space-2xl) on h1–h6.
2.5.8 Target Size (Minimum) Breadcrumb, footer, in-page-nav, and mega-menu link tap areas measured ≤20px tall. All bumped to ≥28px (breadcrumb, footer, in-page-nav) or 36px (mega-menu link).
4.1.3 Status Messages Loading button hid text via color: transparent. SR-only users could see no indication of state. Loading style also triggers on [aria-busy="true"]; component docs require the attribute and an optional .u-visually-hidden "Loading…" text.

v1.2 follow-up audit

A second pass run after the rebrand caught one silent functional regression and a handful of mobile / semantic gaps. All fixed in v1.2:

CriterionFindingFix
Functional regression The rebrand renamed HTML attributes to data-wire-* but missed three dataset.p* reads in js/wire.js. Accordion single-mode and the text-size widget were silently broken. Renamed to dataset.wireAccordion / dataset.wireTextSizeValue. Verified the accordion's one-open-at-a-time behavior and text-size persistence both work end-to-end.
1.1.1 Non-text Content Empty .wire-media placeholders (103 across the site) were exposed to assistive tech as "empty group" with no role or label. Added aria-hidden="true" to every bare .wire-media div. Placeholders that carry a caption stay accessible.
2.4.6 Headings and Labels docs/heroes.html, docs/typography.html, and index.html each rendered multiple <h1>s — one per variant example — fragmenting the document outline. Each page now has exactly one <h1>. Variant examples in .doc-example blocks demote to <h2>; the type-scale row in typography.html uses <p class="wire-h1"> etc. to keep the visual without polluting the outline.
2.5.8 Target Size (Minimum) .wire-tag--sm measured 20px tall, below the AA 24px floor. The .wire-text-size__btn widget — the user-facing accessibility affordance — measured 32px, well under the 44px target appropriate for primary a11y controls. .wire-tag--sm bumped to 24px (AA floor). .wire-text-size__btn bumped to 44px (AAA / Apple HIG) — the widget that exists to help vision-impaired users deserves the strictest target.
1.4.10 Reflow Sticky .wire-inpagenav consumed ~40% of viewport height on a 568px phone. .wire-gallery--mosaic forced horizontal scroll at 320px due to a 6-column grid with multi-span tiles. .wire-help-bar wrapped awkwardly when three children competed for row space on narrow viewports. In-page nav becomes position: static below 48em. Mosaic gallery collapses to a single stacked column below 48em; the asymmetric grid only activates at tablet+. Help bar uses flex-direction: column mobile-first; switches to row at 48em.
4.1.3 Status Messages The text-size widget updated aria-pressed on each button but never announced the new scale to assistive tech — a screen-reader user heard "pressed" with no value context. Added a polite live region (role="status" + aria-live="polite") that announces "Text size: small / medium / large / extra-large" on each change.
Print legibility Collapsed <details> elements lost their content on print — accordion bodies were stripped by the UA stylesheet before the print rules ran. Healthcare users routinely print pre-visit and measure-detail pages. print.css now force-expands every <details> child with display: block !important; and hides the disclosure marker so the printed page reads as flat prose.

Megamenu trigger — deliberate pattern

The mega-menu uses an <a href> as the popup trigger, not a <button>. ARIA APG prefers a button for popup triggers; we accept the trade-off so the menu links remain useful when JavaScript is unavailable or hasn't loaded yet — the parent link navigates to a sensible landing page, and the JS upgrade adds aria-haspopup="true" + aria-expanded and toggle behavior on click. This is the same compromise the W3C describes in the WAI-ARIA "Menubar (Editor)" example for navigation menus.

Out of scope for v1.2

  • Live screen-reader pass with NVDA + VoiceOver. Authored to the patterns; not independently SR-tested yet. Recommended before client launch on any production work.
  • Automated tooling. No axe-core / Pa11y / Lighthouse CI integration yet. Plan: add Pa11y to the Riverside demo pages as a continuous check.
  • WCAG 2.2 AAA. Out of scope by client decision; v1.2 targets AA with selective AAA wins on the text-size widget.
  • Section 508 mapping. Out of scope by client decision; revisit if a federal/public-university engagement lands.
