Tokens

The only file in the system with raw values is css/tokens.css. To rebrand, edit that file. Components reference these variables; nothing else hardcodes.

  • css/tokens.css

Grayscale ramp

Twelve steps, designed so any 50+ value clears 4.5:1 against white, and 40 clears 3:1 (border/large-text only).

gray-00#ffffff— bg
gray-05#fafafa— surface
gray-10#f4f4f4— muted bg
gray-20#e6e6e6— border
gray-30#d4d4d4— inverse text
gray-40#a8a8a83.0:1 / large only
gray-50#7676764.6:1 / body min
gray-60#5959597.1:1 / muted text
gray-70#40404010.4:1
gray-80#2b2b2b14.2:1
gray-90#1a1a1a17.1:1
gray-95#0f0f0f19.0:1 / text

Semantic colors

Always reach for these before the raw ramp. A semantic token explains intent; a ramp value is just a number. Contrast pairings and the AA floor are covered in Accessibility.

--color-text--gray-95Default body and heading text
--color-text-muted--gray-60Lead paragraphs, captions, secondary text
--color-text-subtle--gray-50Placeholders, tertiary meta
--color-border--gray-20Dividers, default borders

--color-accent--gray-95Primary buttons, current-state indicators
--color-focus--gray-95Focus ring

Type scale

Eleven steps. Headings use semibold or bold; body text uses regular at 16px with 1.5 line-height. See Typography for applying the scale — pairing, measure, and when to use each level.

--text-2xs / 12Eyebrow / tag
