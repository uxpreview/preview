Iconography

Icons are monochrome line glyphs, authored inline as SVG and inheriting the text color. There is no icon font or sprite — the set is small and grows only as components need it, but every glyph follows one drawing convention so they read as a family.

Overview

Icons clarify a control or label a status; they never carry meaning on their own. In a grayscale system that means they earn their place by function, not decoration, and they always sit beside or stand in for a word that an assistive technology can read.

  • Inline <svg>, drawn with stroke="currentColor" so an icon takes the color of its surrounding text in either theme.
  • No icon font, no sprite sheet, no build step — an icon is copied into the markup where it's used.
  • One stroke-based style, so a new glyph drawn to the convention drops in without looking foreign.

Drawing convention

Every glyph is drawn on a 24-unit grid with a 2-unit stroke, round caps and joins, and no fill. Keeping the grid and stroke constant is what makes the set feel coherent.

PropertyValueWhy
viewBox0 0 24 24One grid for every glyph; scales cleanly to any display size.
strokecurrentColorInherits text color; works in light and dark with no per-icon override.
stroke-width2Reads at 16–24px without thinning out or going heavy.
linecap / linejoinroundSoft terminals match the system's pill radii and friendly tone.
fillnoneLine style only — no solid shapes, no second color.
A glyph drawn to the convention 
<svg width="24" height="24" viewBox="0 0 24 24" fill="none"
     stroke="currentColor" stroke-width="2"
     stroke-linecap="round" stroke-linejoin="round"
     aria-hidden="true" focusable="false">
  <circle cx="11" cy="11" r="7"></circle>
  <line x1="21" y1="21" x2="16.65" y2="16.65"></line>
</svg>

Sizes

Three render sizes cover the system. The glyph stays a 24 viewBox; only the rendered width/height change, so the stroke scales with it.

16 · 20 · 24
16 20 24
  • 16px — inline with body text and inside dense controls.
  • 20px — default for buttons and form affordances.
  • 24px — standalone controls and the mobile header.
  • An icon-only control keeps a 44×44px hit target regardless of the glyph size — pad the button, don't grow the icon.

Accessible labeling

Whether an icon needs a name depends on its job. Get this wrong and a screen reader either announces nothing useful or repeats the label twice.

Decorative — paired with a visible label 
<button class="wire-button">
  <svg aria-hidden="true" focusable="false">…</svg> Search
</button>
Meaningful — icon-only control needs a name 
<button class="wire-button wire-button--icon-only" aria-label="Open navigation">
  <svg aria-hidden="true" focusable="false">…</svg>
</button>
  • Decorative (a visible word already names the action): mark the SVG aria-hidden="true" focusable="false" so it's skipped.
  • Meaningful (icon stands alone): give the control an aria-label, and still hide the SVG itself.
  • Never rely on the icon alone to convey state — pair it with text, shape, or position (WCAG 1.4.1).

Do / Don't

Do
Draw new glyphs on the 24 grid with a 2-unit stroke so they match the set.
Do
Give every icon-only control an aria-label that names the action.
Don't
Introduce filled, duotone, or colored icons — the system is line-only and grayscale.
Don't
Ship an icon as the sole signal for status or meaning; back it with a word.
Don't
Shrink an icon-only button below the 44px target to fit a smaller glyph.
