# Styles Section Build-out + Consolidation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the Styles section complete and self-contained in `styles/` — relocate the five built pages from `docs/`, build the two Planned stubs (Color & theming, Shape & radius), and normalize all seven so they read as one section.

**Architecture:** The nav is manifest-driven: `manifest.json` → `nav` is the source of truth, and `scripts/build-ia.mjs` regenerates `partials/nav.html` and re-injects nav/footer/shell into every doc-chrome page. So relocation is a `ref` flip + `git mv` + one build run; only non-nav content links are edited by hand. The two new pages are authored to the established built-page anatomy (breadcrumb → `wire-doc-header` → `wire-shell__doc` → `doc-section`s with `doc-example` demos + `wire-table` token tables → `wire-inpagenav` → `wire-doc-pager`), documenting real `css/tokens.css` values, grayscale and tokens-only.

**Tech Stack:** Plain HTML + CSS (no build step at runtime), Node ESM tooling (`build-ia.mjs`), `pa11y-ci` for accessibility, the `preview_*` tools for browser verification.

**Spec:** `docs/superpowers/specs/2026-06-10-styles-section-design.md`

        <section class="doc-section" id="theming">
          <h2 class="wire-h3">The light/dark theming model</h2>
          <p class="u-text-muted">A <code>data-theme</code> attribute on <code>&lt;html&gt;</code> (set by the rail theme toggle, persisted to <code>localStorage</code>) selects the palette. A small inline script in every page's <code>&lt;head&gt;</code> applies it before first paint, so there is no flash. In dark mode, the semantic aliases re-point — components are unchanged because they only ever referenced the semantic layer.</p>
          <div class="wire-table-wrap u-mt-md">
            <table class="wire-table">
              <thead><tr><th scope="col">Mode</th><th scope="col">How it is set</th><th scope="col">What changes</th></tr></thead>
              <tbody>
                <tr><td>Light (default)</td><td><code>color-scheme: light</code> on <code>:root</code></td><td>Semantic tokens alias the light end of the ramp</td></tr>
                <tr><td>Dark</td><td><code>[data-theme="dark"]</code> on <code>&lt;html&gt;</code></td><td>Semantic aliases flip to the dark end; component CSS untouched</td></tr>
                <tr><td>System</td><td><code>prefers-color-scheme</code> via the toggle's "system" option</td><td>Follows the OS setting</td></tr>
              </tbody>
            </table>
          </div>
          <p class="u-text-muted u-mt-md">Use the theme toggle at the bottom of the left rail to see every swatch on this page re-theme live — they are all driven by tokens, so nothing on the page is hardcoded to a mode.</p>
