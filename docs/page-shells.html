Pattern

Page shells

Four scaffolds that hold the system's components in the right order. Not classes — page recipes, with the same skip link, top nav, and footer at every entry point, and a predictable middle that varies by purpose.

What every shell shares

Every page in the system, regardless of shell, starts with a skip link, a top nav, and a <main id="main">; ends with a footer; and links the system stylesheet and behavior script.

Common scaffold
.wire-skip-link (visually hidden until focus)
.wire-topnav (sticky brand · primary nav · utility)
<main id="main"> — shell-specific content
.wire-footer (compact, standard, expanded, or inverse)

1. Landing shell

The marketing front door. Hero leads, then a sequence of content blocks (feature grid, two-column, callout, quote, timeline), then a closing CTA in the expanded footer.

Region order
.wire-topnav
.wire-hero · editorial / split / centered / stat-led
u-section · stat strip (.wire-card--stat in u-grid)
u-section · feature grid (.wire-feature-grid--3)
u-section · two-column highlight (.wire-twocol)
u-section · article teasers (.wire-card grid)
u-section · callout or quote moment
.wire-footer.wire-footer--expanded · closing CTA

When to use

  • Homepage, program landing pages, campaign pages.
  • Any page where the goal is awareness and one clear next step.

When not to use

  • For long-form reading — use the article shell.
  • For a directory or index — use the listing shell.

2. Article shell

Long-form editorial. Breadcrumb above the article header, a hero with eyebrow and lead, a sidebar with in-page nav for sections, and a prose container holding the body.

Region order
.wire-topnav
u-section · .wire-breadcrumb
.wire-hero · editorial (eyebrow · title · lead · meta)
.wire-inpagenav (sticky) .wire-prose · article body, .wire-quote, .wire-callout, .wire-media, .wire-timeline
u-section · related articles (.wire-article-list)
.wire-footer

When to use

  • News, research write-ups, patient stories, principle and policy pages.
  • Anything where a table of contents helps the reader orient.

When not to use

  • Short pages without sections — drop the in-page nav and use the landing shell.
  • FAQ-style pages — reach for the accordion instead.

3. Listing shell

A filterable index. Hero introduces the index, a filter row (tags or selects) lives above the results, and a grid of cards or article list fills the body. Pagination at the bottom.

Region order
.wire-topnav
u-section · .wire-breadcrumb
.wire-hero · centered (title · short lead)
u-section · search (.wire-search) + filter tags (.wire-tag-list)
.wire-sidenav · faceted filters .u-grid--3 · .wire-card · or .wire-article-list
u-section · .wire-pagination
.wire-footer

When to use

  • Directories — conditions, providers, research labs, publications, news.
  • Resource libraries and case study indexes.

When not to use

  • When fewer than six items exist — a single feature grid is enough.
  • When items have rich detail that demands a full card per row — use the article list.

4. Detail shell

A single entity: a provider, condition, lab, or program. Split hero pairs identity with a media moment; the body uses two-column and feature grids to expose facets; a closing CTA pushes to appointment or contact.

Region order
.wire-topnav
u-section · .wire-breadcrumb
.wire-hero · split (text + media · primary CTA)
u-section · stat strip (.wire-card--stat · key metrics)
.wire-inpagenav .wire-tabs or stacked u-section blocks · two-column, accordion, .wire-deflist
u-section · related people (.wire-card--person grid) or articles
u-section · .wire-callout--inverse · closing CTA
.wire-footer

When to use

  • Provider profiles, condition pages, lab pages, program pages, research study pages.
  • Anywhere the page represents one nameable thing.

When not to use

  • When the entity has only a few facts — a single article shell is leaner.
  • When the page is mostly editorial narrative — use the article shell.

Accessibility for every shell

  • Every shell starts with a .wire-skip-link so keyboard users can jump past the top nav.
  • The page h1 lives once, inside the hero. Subsequent sections use h2 and below.
  • Each <nav> has a unique aria-label — Primary, Breadcrumb, On this page, Pagination.
  • <main id="main"> is the skip target and the primary landmark; only one per page.
  • The footer is always inside <footer> and outside <main>.
