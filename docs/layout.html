Layout

The structural primitives that compose every page — container, stack, cluster, grid, sidebar, and friends. Each does one job, leans on gap-based spacing, and stays responsive without bespoke CSS, so components can stay focused on identity.

Container

Centers content with a responsive max width. Override per-instance with the --container-max custom property or the size modifiers.

.u-container — default (wide)
Wide container — 75rem max-width
Size modifiers
.u-container--narrow · 40rem · article body widths
.u-container--md · 54rem · long-form pages
.u-container--wide · 75rem · default marketing pages
.u-container--full · 90rem · dashboards

Sidebar

Two-track layout: fixed-width sidebar, fluid main. Wraps gracefully on narrow screens.

Vertical padding for page sections. Use size modifiers --sm, --md, --lg, --xl.

.u-section

Spacing

Every gap and inset comes from one 4px-based scale, applied through the primitives above — a stack's or cluster's gap — rather than ad-hoc margins. The full visual scale lives in Tokens; the steps and their typical use:

TokenValueTypical use
--space-3xs2pxHairline gaps, icon nudges
--space-2xs4pxTight chip / label gaps
--space-xs8pxInline clusters, button groups
--space-sm12pxCompact stacks, list rows
--space-md16pxDefault element spacing
--space-lg24pxGroup separation
--space-xl32pxSub-section breaks
--space-2xl48pxSection padding
--space-3xl64pxMajor section breaks
--space-4xl96pxHero / landing rhythm
--space-5xl128pxFull-bleed spacing

Breakpoints

Four min-width breakpoints, set in em so they respect the reader's text size. The primitives respond at these points; most components stay fluid below them.

NameMin widthTypical shift
sm30em · 480pxStacks loosen; small grids go multi-column
md48em · 768pxSidebars appear; two-column layouts
lg64em · 1024pxPersistent rail; full grids
xl80em · 1280pxDoc TOC column; widest containers
