Navigation

Top nav, mega menu, sidebar nav, in-page nav, breadcrumb, mobile drawer, and pagination. Each handles a different orienting question — where am I, where can I go, how do I get back.

Anatomy

A site typically uses a top nav across all pages, a breadcrumb on detail pages, in-page nav on long-form content, and a footer-anchored drawer for mobile. Pagination and sidebars appear on listing and section pages.

When to use

  • Use top nav as the always-visible primary navigation across every page.
  • Use breadcrumb on pages two or more levels deep so users can step back without the back button.
  • Use in-page nav when a long article has three or more named sections.
  • Use sidebar nav on documentation, account, or settings pages with a stable secondary structure.
  • Use pagination when a list crosses one screen and infinite scroll would obscure totals.

When not to use

  • Do not duplicate the same links in top nav, breadcrumb, and sidebar — choose the primary surface for each.
  • Do not put a mega menu on a small site; a flat top nav is faster to scan.
  • Do not use breadcrumbs on a homepage or single-level site.

Accessibility

  • Wrap every navigation in <nav> with an aria-label distinguishing it (Primary, Breadcrumb, Pagination, On this page).
  • Mark the current item with .is-current and aria-current="page" (or "step", "location" as appropriate).
  • The mega menu opens with click and Escape closes it; keyboard focus moves into the panel on open.
  • Pagination uses aria-disabled on inactive prev/next; the current page is a non-link span.
  • Mobile drawer is wired by data-wire-drawer-open/data-wire-drawer-close; focus is trapped while open and restored on close.
