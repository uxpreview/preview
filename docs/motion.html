Motion

Motion in the system is quiet and functional. It clarifies a change of state — a panel opening, a drawer sliding in — and never decorates. Three durations and two easing curves cover everything, and all of it collapses to nothing when the reader asks for reduced motion.

Tokens

Every transition references one of three duration tokens and one of two easing curves. Components never hardcode a timing value.

TokenValueUse
--duration-fast150msMicro-interactions: hover, focus, toggle, small fades.
--duration-base250msThe default — reveals, accordions, tab swaps.
--duration-slow400msLarger moves: drawers, modals, full overlays.
--ease-outcubic-bezier(0.16, 1, 0.3, 1)Things entering — quick start, soft landing.
--ease-in-outcubic-bezier(0.4, 0, 0.2, 1)Things moving in place or both entering and leaving.

When to use which

Match the duration to the size of the change: the bigger the element or the distance it travels, the slower it should move so the eye can follow it.

  • Fast (150ms) — a state change in place: a button darkening on hover, a tag toggling selected, a focus ring appearing.
  • Base (250ms) — a small reveal: an accordion panel, a tab's content, a tooltip.
  • Slow (400ms) — a large element travelling across the screen: a drawer sliding in, a modal and its backdrop.
Fast, in place — hover or focus the button

Reduced motion is a contract

When the operating system reports prefers-reduced-motion: reduce, every duration token drops to 0ms at the source. Transitions become instant; nothing animates. Because components reference the tokens, this is automatic — there is nothing to remember per component.

The contract, in tokens.css 
@media (prefers-reduced-motion: reduce) {
  :root {
    --duration-fast: 0ms;
    --duration-base: 0ms;
    --duration-slow: 0ms;
  }
}

Motion must never be the only way a state is conveyed, and nothing flashes or auto-plays (WCAG 2.2.2, 2.3.3). If you write a custom animation, drive its duration from a token so it honors this contract too.

In context

Where the tokens show up across the system. Each is a state change made legible, never an effect.

ComponentMotionToken
Button / TagBackground and border ease on hover and focus.fast · ease-out
Accordion / TabsPanel height and content settle on open.base · ease-out
Tooltip / ToastOpacity fades in and out.base · ease-out
