System reference

Resources

The working reference behind the system: the utility helpers, the evidence record that defends each decision, the validation protocol, and the component stability map. Not design foundations — the methodology and tooling that keep them honest.

Utilities

Single-purpose helper classes — text, background, visibility, and one-off width and spacing — for the cases a layout primitive doesn't cover.

Evidence model

The four-tier basis — standard, empirical, convention, judgment — behind each decision, cited component by component.

Validation

What we test and how — accessibility, performance, browser and print matrices — and an honest account of what isn't wired up yet.

Component status

What's safe to build on and what's still settling — stable, beta, alpha, deprecated — across every component.
