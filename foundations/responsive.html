Responsive

Every page is built mobile-first and reflows fluidly. There is one set of breakpoints, taken from tokens, and the same content has to work from a 320 px phone to a wide desktop — without a second layout to maintain.

Breakpoints

Four named breakpoints, defined as constants in css/tokens.css and expressed in em so they scale with the reader's text size rather than a fixed pixel grid.

NameMin widthTypical shift
sm30em / 480pxLarge phones — cards can sit two-up.
md48em / 768pxTablets — the left rail appears, columns split.
lg64em / 1024pxLaptops — full multi-column layouts.
xl80em / 1280pxWide desktop — content width holds; margins grow.
Why em, not px

Because the breakpoints are em-based, a reader who bumps their browser text size up moves the layout with them — the page reflows to a more spacious arrangement instead of cramming larger type into a desktop grid.

Mobile-first

Base styles target the narrow viewport; min-width media queries layer enhancements on top. The default is a single column, stacked vertically, with full-width controls. Wider screens add columns and horizontal structure — they never strip anything away.

This order matters: the smallest screen is the hardest constraint, so it is designed first. If the content works there, widening the viewport only ever gives it more room.

Fluid by default

Between breakpoints, the layout flexes. Containers cap and center the content; spacing comes from the token scale so gaps grow with the page rather than snapping at fixed sizes.

  • u-container — the standard reading-width wrapper for article and form content.
  • u-container--wide — a wider wrapper for landings, listings, and dashboards.

Avoid fixed pixel widths on content blocks. Let the container set the bounds and let the element fill the space it is given.

What changes, what doesn't

Layout adapts; the accessibility floor does not. Targets, focus, and contrast are the same at every width.

RegionNarrowWide
NavigationOff-canvas drawer behind a burger.Persistent left rail.
ColumnsStacked, full width.Multi-column grid.
TablesScroll inside wire-table-wrap.Sit at full width.
Tap targets≥44px.≥44px — unchanged.

Testing the range

Before a page is done, walk it across the range. A quick checklist:

  • At 320px wide there is no horizontal scroll and nothing is clipped.
  • At 200% browser zoom the content still reflows to one column with no loss of function (WCAG 2.2 — 1.4.10 Reflow, a requirement).
  • The rail collapses to the drawer below md, and the drawer traps focus while open.
  • Touch targets hold at ≥44px on the phone layout.
  • Reduced-motion preferences are respected for any layout transition.
