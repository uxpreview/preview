Divider

Stable

A thin rule that separates content. Horizontal for a thematic break between blocks, vertical for inline items, with weight and style variants and an optional centered label.

  • css/components/divider.css
  • Atom · no JavaScript

Overview

A divider draws a line between things. Most separation should come from spacing; reach for a rule only when a visible break genuinely helps the reader tell two groups apart.

When to use

  • A thematic break between sections of a long page.
  • An "or" between two alternatives (the labeled divider).
  • A vertical rule between inline actions or metadata in a row.

When not to use

  • Where whitespace already groups the content. A rule then adds noise.
  • To box every item in a list; let the list's own rhythm carry it.
  • As decoration with no structural meaning; if it is purely visual, mark it aria-hidden.

Anatomy

A single rule, drawn as an <hr> for a semantic break. The labeled form is a separate wrapper that draws a line on each side of a label.

PartClassRole
Rule.wire-dividerThe line. An <hr> for a real break; a decorative element if purely visual.
Labeled.wire-divider-labeledA flex wrapper that puts a line on either side of a centered label.

Behaviors

Divider is static and non-interactive: no states, no hover or focus, no JavaScript. The variants (see Specs) are its only variation.

Composability

  • Standalone — the plain rule takes no content; place it between blocks.
  • Labeled — .wire-divider-labeled wraps a short label; the lines are drawn for you.
  • Inline — a vertical rule sits between inline items inside a flex row.

Item

Content guidelines

  • A labeled divider holds one short word, usually "or"; keep it lowercase or small caps.
  • Do not write a sentence into a divider label; it is a connector, not a heading.

Composition patterns

  • Section breaks — a single rule between long-form sections.
  • Auth alternatives — an "or" divider between sign-in methods.
  • Inline toolbars — vertical rules between grouped actions.
  • Card footers — a strong rule above a footer action row.

States

None. Divider has no interactive states; its visual set is the variants in Usage › Types & variants.

JavaScript

None. Divider is pure CSS with no data-wire-* hooks.

Code

<!-- Horizontal (semantic break) -->
<hr class="wire-divider">
<hr class="wire-divider wire-divider--strong">
<hr class="wire-divider wire-divider--dashed">

<!-- Labeled -->
<div class="wire-divider-labeled">or</div>

<!-- Vertical (decorative, between inline actions) -->
<span class="wire-divider wire-divider--vertical" aria-hidden="true"></span>

ARIA & semantics

  • For a real thematic break, use <hr class="wire-divider">. It carries an implicit separator role, so the break is conveyed, not just drawn.
  • For a purely decorative rule that adds no structure, use a non-semantic element and mark it aria-hidden="true" (the vertical rule between inline actions is usually this case).
  • If a vertical rule does separate two meaningful groups, use role="separator" aria-orientation="vertical" instead of hiding it.
  • The labeled divider's text is real content; keep it in the element so it is read.

Keyboard interactions

None. A divider is not interactive and is not in the tab order.

WCAG 2.2 criteria mapped

SCHow Divider satisfies it
1.3.1 Info & RelationshipsA semantic <hr> conveys the break programmatically; decorative rules are hidden so they add no false structure.
1.4.1 Use of ColorNo meaning rests on the line's color; grayscale, and any real grouping is also structural.
1.4.11 Non-text ContrastThe default rule is decorative, so the 3:1 boundary requirement does not apply; use --strong when a visible separation must carry weight.

Screen reader notes

  • An <hr> is announced as a separator, giving a sense of the section break.
  • A decorative rule marked aria-hidden is skipped entirely, which is correct when it carries no meaning.
  • The labeled divider reads its label inline (for example, "or").
