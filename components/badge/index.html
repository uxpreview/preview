Badge

Stable

A small, non-interactive label that annotates another element with status or category. Four emphasis variants by fill density, an optional leading dot, and a count overlay for notifications.

  • css/components/badge.css
  • Atom · no JavaScript

Overview

A badge labels something else: a status on a card, a category in a list, a count on an icon. It is read, not operated. Keep it to a word or two, and use it sparingly so each one still signals.

When to use

  • Status on an entity: "Accepting new patients", "In range", "Confirmed".
  • A short category or type label attached to a card or row.
  • An unread count on an icon or nav item (the count overlay).

When not to use

  • Do not make it clickable or removable. If the user acts on it, use a Tag instead.
  • Do not use it for the maturity of a documented component; that is the doc-only Status-pill.
  • Do not lean on the dot alone to carry meaning; always pair it with a text label.

Anatomy

A pill-shaped span holding a short label, with an optional leading dot. The count overlay is the same span pinned to the corner of a host wrapped in an anchor.

block · dot · label Label
PartClassRole
Block.wire-badgeThe <span> itself; holds the label and carries the emphasis variant.
Dot.wire-badge__dotOptional 0.5rem leading dot in currentColor. Decorative; the label carries the meaning.
Count.wire-badge--countOverlay form: a filled circle pinned to a host's corner for a notification count.
Anchor.wire-badge-anchorWraps the host so the count overlay can position to its corner.

Types & variants

One structural type. In grayscale, fill density does the differentiating; the text label is always load-bearing.

Behaviors

Badge is static and non-interactive by design: no states, no hover or focus, no JavaScript. Its visual variants (see Specs) carry the differentiation that states carry on interactive components. When a badge's value changes at runtime, for example a count, wrap it in an aria-live region so the change is announced (see Accessibility).

Composability

Badge is a leaf. It composes into other components rather than holding content of its own.

  • Label — the only slot: one to two words of text, optionally preceded by a .wire-badge__dot.
  • On cards and rows — drop a badge into a card body or a table cell to mark status.
  • As a count — wrap a host (icon button, avatar) in .wire-badge-anchor and add a .wire-badge--count.

Do / Don't

Do

In range

One short status, with the meaning in the words. The dot is a quiet accent.

Don't

New Updated Popular Hot

Stacking badges dilutes every one of them. Pick the single status that matters.

Content guidelines

  • One to two words. The badge is small; a sentence does not belong in it.
  • Name the state plainly: "In range", "Flagged", "Confirmed".
  • Front-load the meaningful word; the label is read on a quick scan.
  • For counts, show the number; cap overflow as "9+" rather than a long figure.

Composition patterns

  • Provider / entity cards — a status badge in the card body.
  • Result tables — "In range" / "Flagged" in a status column.
  • Toolbars and nav — a count overlay on a notifications or messages icon.
  • Filters — do not use Badge here; an interactive filter chip is a Tag.

States

Badge has no interactive states (no hover, focus, active, or disabled). Its visual set is the variants in Usage › Types & variants, whose fills map to the tokens above.

JavaScript

None. Badge is a static <span> with no data-wire-* hooks. The count overlay needs no script either; its accessible name is authored on the host. If a count updates at runtime, the application wraps the host in an aria-live="polite" region.

Code

<!-- Variants -->
<span class="wire-badge">Confirmed</span>
<span class="wire-badge wire-badge--subtle">Flagged</span>
<span class="wire-badge wire-badge--outline">Draft</span>
<span class="wire-badge wire-badge--inverse">Urgent</span>

<!-- With a leading dot -->
<span class="wire-badge">
  <span class="wire-badge__dot" aria-hidden="true"></span>Accepting new patients
</span>

<!-- Count overlay: host carries the accessible name, count is aria-hidden -->
<span class="wire-badge-anchor">
  <button class="wire-button wire-button--icon-only wire-button--ghost"
          type="button" aria-label="Notifications, 3 unread">
    <svg aria-hidden="true" focusable="false">...</svg>
  </button>
  <span class="wire-badge wire-badge--count" aria-hidden="true">3</span>
</span>

ARIA & semantics

  • Badge is a non-interactive <span>. It needs no role and is not focusable.
  • Meaning lives in the text label, never in fill or the dot alone. The __dot is decorative (aria-hidden).
  • Count overlay: put the count in the host control's accessible name (e.g. aria-label="Notifications, 3 unread") and mark the visual count aria-hidden="true" so it is announced once, unambiguously.
  • If a count changes at runtime, wrap the host in an aria-live="polite" region.

Keyboard interactions

None. Badge is not interactive and is not in the tab order. The host of a count overlay (a button) is what receives focus and keyboard, per its own component. For an interactive, removable chip, use Tag instead.

WCAG 2.2 criteria mapped

SCHow Badge satisfies it
1.4.1 Use of ColorMeaning is in the text label, not hue. Grayscale guarantees no color-only information.
1.4.3 Contrast (Minimum)Measured: default 17.4:1, subtle 6.7:1, outline 19.2:1, inverse 19.2:1. All clear 4.5:1.
1.4.11 Non-text ContrastThe outline border and the count's ring are decorative; the label, not the border, conveys the status, so no 3:1 boundary is relied upon.
4.1.2 Name, Role, ValueA labeled span needs no role. For counts, the host's accessible name carries the value and the visual is aria-hidden.

Screen reader notes

  • An inline badge is read as plain text in the flow, e.g. "Accepting new patients".
  • A count overlay reads through the host: "Notifications, 3 unread, button". The visual "3" is not double-announced because it is aria-hidden.
  • Do not rely on the dot; a screen reader skips it, so the label must stand alone.
  • Runtime count changes are announced only inside an aria-live region; the badge itself does not announce.
