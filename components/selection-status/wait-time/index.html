Wait time

Beta

A current wait estimate, shown plainly: a label, one large tabular figure with its unit, and a note saying how fresh it is. Built for emergency-department and urgent-care headers, where a clear number does more to settle a worried visitor than any reassurance.

  • css/components/wait-time.css
  • Molecule · no JavaScript

Overview

Wait time answers one anxious question — "how long?" — with a single legible number. Showing the estimate at all is the point: a visible, honestly-dated figure manages expectations far better than silence. The component keeps three things together: what is being measured, the figure itself, and how current it is.

When to use

  • Emergency-department and urgent-care headers and location pages.
  • A row in a list of nearby locations, comparing current waits.
  • Callback or appointment estimates — "next available", "typical hold".

When not to use

  • Don't show a wait you can't keep reasonably fresh; a stale figure erodes trust faster than none.
  • For a categorical state ("Open", "Closed"), use a Status dot with a label, not a number.
  • For a generic count or metric unrelated to waiting, use a KPI stat.

Anatomy

A label on top, a large tabular figure with its unit, and a freshness note beneath. The figure dominates; the label and note give it meaning and credibility.

Types & variants

One structure, three sizes for three placements.

VariantSelectorUse
Block.wire-wait-timeDefault. A stacked figure for a header or card.

Large.wire-wait-time--lgAn oversized figure for a hero or stat band.

Composability

Wait time owns the figure; presence and surrounding structure come from its neighbours.

  • With Status dot — a leading dot in the label shows open/closed alongside the wait.
  • In a card or hero — the block or large form anchors a location header.
  • In a list row — the inline form compares waits across locations.

Do / Don't

Do

Current ER wait 12 min Updated 4 min ago

Name what it measures and date it. The freshness note is what makes the number trustworthy.

Don't

12

A bare number with no label or date answers nothing — 12 of what, measured when?

Content guidelines

  • Always pair the figure with a label and a freshness note.
  • Round to a sensible unit — whole minutes, not "12.4".
  • Say it's an estimate when it is; don't imply precision you don't have.
  • Keep the label short and concrete: "Current ER wait", not "Approximate current waiting period".

States

No interactive states. The figure is read, not operated. Its visual set is the three size variants in Usage › Types & variants; presence comes from a composed Status dot, not a state here.

JavaScript

None in the component. The figure and its freshness note are supplied and refreshed by the application; when the value updates at runtime, wrap the block in an aria-live="polite" region so the new wait is announced. No data-wire-* hooks.

Code

<!-- Block (default). Wrap in aria-live if it updates at runtime. -->
<span class="wire-wait-time" aria-live="polite">
  <span class="wire-wait-time__label">Current ER wait</span>
  <span class="wire-wait-time__amount">
    <span class="wire-wait-time__value">12</span>
    <span class="wire-wait-time__unit">min</span>
  </span>
  <span class="wire-wait-time__meta">Updated 4 min ago</span>
</span>

<!-- Label with a presence dot from Status dot -->
<span class="wire-wait-time__label">
  <span class="wire-status-dot" aria-hidden="true"></span>Riverside ER · Open 24 hours
</span>

<!-- Inline (list row) and large (hero) -->
<span class="wire-wait-time wire-wait-time--inline">…</span>
<span class="wire-wait-time wire-wait-time--lg">…</span>

ARIA & semantics

  • Keep the label, value, unit, and meta as adjacent text so the block reads as one phrase: "Current ER wait, 12 min, updated 4 min ago".
  • Don't split the number from its unit or label with anything a screen reader announces between them.
  • A leading status dot in the label is decorative (aria-hidden); its meaning is in the label text.
  • For a live figure, wrap the block in aria-live="polite" so an update is announced once, calmly.
  • If "min" is too terse, expand it for assistive tech (e.g. aria-label on the amount reading "12 minutes").

Keyboard interactions

None. Wait time is read-only and not in the tab order. If it sits inside a link to a location page, that link owns the keyboard behaviour and focus ring.

WCAG 2.2 criteria mapped

SCHow Wait time satisfies it
1.4.1 Use of ColorThe figure and its label carry everything; no meaning rides on colour. A composed status dot still depends on its text label.
1.4.4 Resize TextAll sizes are token-based (rem type scale) and reflow without clipping when text is enlarged.
1.4.10 ReflowThe inline variant wraps; the block variant stacks — neither needs horizontal scrolling.
4.1.3 Status MessagesRuntime updates are announced through an aria-live region wrapping the block.

Screen reader notes

  • Read in order, the block announces as "Current ER wait, 12 min, updated 4 min ago" — keep the DOM order label → value → unit → meta.
  • "min" is typically read as "min"; expand to "minutes" via aria-label if a clearer reading matters.
  • Inside an aria-live="polite" region, a change from 12 to 15 is announced once, without stealing focus.
  • The decorative presence dot is skipped; the open/closed wording in the label is what is read.
