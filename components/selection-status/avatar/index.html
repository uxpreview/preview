Avatar

Beta

A small thumbnail standing in for a person or entity. Three fallbacks in priority order — image, initials, then a generic placeholder icon — in four sizes and two shapes. The foundation that Avatar group, Byline, and person cards build on.

  • css/components/avatar.css
  • Atom · no JavaScript

Overview

An avatar puts a face — or a stand-in for one — next to a name, so a person or organisation is recognisable at a glance. In a grayscale wireframe the photo rarely exists yet, so the avatar usually renders as initials or a placeholder icon; the image slot is wired and ready for higher-fidelity stages.

When to use

  • Beside a person's name: a provider in a directory, an author byline, a comment.
  • As a compact identity in a card, list row, or table cell.
  • Stacked into an Avatar group to show a care team or set of contributors.
  • As a host for a count or status overlay (wrap it in a Badge anchor).

When not to use

  • Don't use it as a decorative spot illustration; that is Media.
  • Don't put a multi-word label inside it — initials are one or two letters. Longer labels belong in a Badge or Tag.
  • Don't rely on the avatar alone to identify someone; always pair it with the visible name.

Anatomy

A fixed-size circle that centres exactly one of three things: an image, initials text, or a placeholder icon. Pick the richest one available and fall back down the list.

Composability

Avatar is a leaf that other components lean on. It composes into larger parts rather than holding content of its own.

  • Into Avatar group — overlapping avatars with a ring show a team or set of contributors.
  • Into Byline and person cards — the avatar sits beside the name and role.
  • As a Badge host — wrap it in .wire-badge-anchor and pin a .wire-badge--count for an unread count.

Don't

Content guidelines

  • Initials are one or two letters — typically first + last. Avoid three.
  • Derive initials from the displayed name so they always match what the user reads.
  • Use the placeholder icon, not random initials, when no name is known.
  • Reserve the square shape for non-human entities; people are circles.

Measurements & structure

Like Badge's dot and count, the avatar's diameters are fixed component geometry expressed in rem via the local --avatar-size custom property, not spacing tokens.

PropertyValue
Size — sm / md / lg / xl2rem · 2.5rem · 3.5rem · 5rem (--avatar-size).
ImageFills 100% and crops with object-fit: cover; clipped by overflow: hidden.
Icon60% of the circle, --color-text-muted.

Ringbox-shadow ring at --border-md in --color-bg; adds no layout box.

States

Avatar has no interactive states of its own (no hover, focus, active, or disabled). When it is the content of a link, the focus ring and hover belong to that link, per its own component. Its visual set is the sizes and shapes in Usage › Types & variants.

JavaScript

None. Avatar is static markup with no data-wire-* hooks. Choosing the fallback (image vs initials vs icon) and deriving initials from a name are the application's job; the component only renders what it is given.

Code

<!-- Image (richest fallback). Empty alt when the name is shown beside it. -->
<span class="wire-avatar">
  <img class="wire-avatar__img" src="elena-ruiz.jpg" alt="">
</span>

<!-- Initials. Decorative when the name is adjacent. -->
<span class="wire-avatar" aria-hidden="true">ER</span>

<!-- Placeholder icon (quietest fallback, always decorative). -->
<span class="wire-avatar" aria-hidden="true">
  <svg class="wire-avatar__icon" aria-hidden="true" focusable="false">...</svg>
</span>

<!-- Sizes and the square entity shape -->
<span class="wire-avatar wire-avatar--sm">ER</span>
<span class="wire-avatar wire-avatar--xl">ER</span>
<span class="wire-avatar wire-avatar--square">RH</span>

<!-- Standalone, no visible name: the avatar carries the name -->
<a class="wire-avatar" href="/team/elena-ruiz" aria-label="Dr. Elena Ruiz">ER</a>

<!-- As a count host (overlay from Badge) -->
<span class="wire-badge-anchor">
  <a class="wire-avatar" href="/messages" aria-label="Dr. Elena Ruiz, 2 new messages">ER</a>
  <span class="wire-badge wire-badge--count" aria-hidden="true">2</span>
</span>

ARIA & semantics

The accessible name follows one rule: name the person once. Whether that happens on the avatar or beside it decides how the avatar is exposed.

  • Name shown beside the avatar (the common case) — the avatar is decorative. An image takes alt=""; initials or an icon avatar takes aria-hidden="true" so "ER" is not spelled out.
  • Avatar stands alone (no visible name) — it must carry the name itself: a descriptive alt on the image, or aria-label on the link/element holding initials.
  • The placeholder icon is always decorative — aria-hidden="true", focusable="false".
  • Initials are an abbreviation, not the name; never let them be the only accessible identity.

Keyboard interactions

None of its own. A bare avatar is not in the tab order. When wrapped in a link or button it inherits that control's keyboard behaviour and focus ring; the avatar adds nothing and removes nothing.

WCAG 2.2 criteria mapped

SCHow Avatar satisfies it
1.1.1 Non-text ContentImage avatars take a text alternative — empty when the name is adjacent, descriptive when the avatar stands alone. Icon and initials fallbacks are marked decorative.
1.4.1 Use of ColorIdentity is the name and the shape, never hue. Grayscale guarantees no colour-only meaning.
1.4.3 Contrast (Minimum)Default initials are --color-text on --color-bg-muted, which clears 4.5:1. The icon is decorative, so the 3:1 graphic floor is not relied upon.
4.1.2 Name, Role, ValueA standalone avatar exposes a name via alt/aria-label; a decorative one is hidden so the adjacent name is announced once.

Screen reader notes

  • In the common "avatar + name" pattern only the name is announced; the decorative avatar is skipped.
  • Unhidden initials are read letter by letter ("E, R"), which is why decorative initials must be aria-hidden.
  • A standalone avatar link reads its accessible name and role, e.g. "Dr. Elena Ruiz, link".
  • A count overlay is announced through the host's name (see Badge), e.g. "Dr. Elena Ruiz, 2 new messages, link".
