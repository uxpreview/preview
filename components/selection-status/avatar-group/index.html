Avatar group

Beta

Overlapping avatars that summarise a set of people at a glance — a care team, a list of contributors — with an optional "+N" tile when there are more than fit. Composes Avatar children; it adds the overlap and the ring, nothing else.

  • css/components/avatar-group.css
  • Molecule · no JavaScript

Overview

An avatar group is a glanceable summary of who is involved, not a roster. It overlaps a handful of avatars to signal "a group of people" and caps the visible set with a "+N" tile. The authoritative, named list lives elsewhere — a link, a roster, a tooltip.

When to use

  • Show a care team, project team, or set of authors compactly.
  • Signal participation — who has commented, joined, or contributed.
  • Preview a larger set, with a "+N" tile linking to the full list.

When not to use

  • Don't use it as the only way to reach people; pair it with a real, named list or link.
  • Don't overlap more than about five before switching to "+N"; past that it is noise.
  • Don't mix sizes within one group — keep every avatar the same size.

Anatomy

A flex row of .wire-avatar children with a negative inline margin so each tucks under the previous. The group paints a ring on every child; the last child is often the overflow tile.

avatar · avatar · more ER JM +2
PartClassRole
Group.wire-avatar-groupThe flex row. Owns the overlap and rings its children.
Member.wire-avatarAny avatar — image, initials, or icon — at one shared size.
Overflow.wire-avatar-group__moreA trailing "+N" tile, a quieter avatar standing for the rest.

Sizes & overflow

The group has no size of its own; it inherits from the avatars inside it. Set the same size modifier on every member. The overflow tile is optional.

small · medium · large
ER JM AO +1 ER JM AO +1 ER JM AO +1

Composability

Avatar group is a thin wrapper: it composes Avatar and adds two things — the overlap and a ring on each child. Everything else (size, shape, image vs initials) is the avatar's own.

  • Members — any .wire-avatar; keep them one size and, for people, circular.
  • Overflow — a final .wire-avatar-group__more tile for the remainder.
  • Into cards and rows — drop a group into a project card or a list row to show participants.

Do / Don't

Do

ER JM AO +5

Cap the visible set at a few and let "+N" carry the rest. The group stays legible.

Don't

ER JM AO RH

Don't mix sizes or show every member. Uneven tiles and a long row both read as clutter.

Content guidelines

  • Show three to five faces, then "+N" for the remainder.
  • The "+N" number is the count not shown, so visible + N equals the total.
  • Give the group a plain summary name: "Cardiology care team, 12 people".
  • Always offer a way to the full named list nearby ("Meet the team").

Tokens used

TokenPurpose
--space-smThe overlap amount (applied as a negative inline margin via --avatar-group-overlap).
--border-mdRing thickness on each member.
--color-bgRing colour — the surface, so it reads as a cut between tiles.
--color-bg-subtleThe overflow tile's fill, quieter than a person.
--color-text-mutedThe overflow tile's "+N" text.
--text-2xsThe overflow tile's type size.

Member sizing, shape, and fill come from Avatar's tokens, not this component.

Measurements & structure

PropertyValue
Layoutdisplay: inline-flex; members aligned centre.
Overlapmargin-inline-start: calc(-1 * --avatar-group-overlap) on every member after the first.
Ringbox-shadow ring at --border-md in --color-bg on each member; no layout box added.
StackingLater members paint over earlier ones (DOM order); the ring keeps each readable.
Overflow tileA .wire-avatar with a subtle fill and muted "+N" text; same size as its peers.

States

No interactive states. If members are links, each carries its own hover and focus per Avatar. The group has no hover, focus, or selected state of its own.

JavaScript

None. Deciding how many avatars to show and computing the "+N" remainder is the application's job; the component renders the row it is given. No data-wire-* hooks.

Code

<!-- Group summary on the wrapper; members are decorative;
     the full named list lives in the adjacent link. -->
<span class="wire-avatar-group" role="group" aria-label="Care team, 9 people">
  <span class="wire-avatar" aria-hidden="true">ER</span>
  <span class="wire-avatar" aria-hidden="true">JM</span>
  <span class="wire-avatar" aria-hidden="true">AO</span>
  <span class="wire-avatar" aria-hidden="true">RH</span>
  <span class="wire-avatar wire-avatar-group__more" aria-hidden="true">+5</span>
</span>
<a href="/care-team">View care team</a>

<!-- One shared size on every member -->
<span class="wire-avatar-group" role="group" aria-label="Authors, 2 people">
  <span class="wire-avatar wire-avatar--sm" aria-hidden="true">ER</span>
  <span class="wire-avatar wire-avatar--sm" aria-hidden="true">JM</span>
</span>

ARIA & semantics

Treat the group as a summary with one accessible name, and route to the real people elsewhere. Spelling out four sets of initials helps no one.

  • Give the wrapper role="group" and an aria-label stating what it is and how many: "Cardiology care team, 12 people".
  • Mark each member avatar and the "+N" tile aria-hidden="true" — the initials are decorative inside the summary.
  • Provide a real, named destination nearby (a link or list). The group previews; the link delivers.
  • If members must be individually actionable, make each a labelled link and drop the blanket aria-hidden — but prefer the summary pattern.

Keyboard interactions

None for the group. In the summary pattern the only focus stop is the adjacent "view all" link. If members are individually linked, they enter the tab order in DOM order, each with its own focus ring per Avatar.

WCAG 2.2 criteria mapped

SCHow Avatar group satisfies it
1.1.1 Non-text ContentMember avatars are decorative within the summary (aria-hidden); the group's name and the linked list carry the information.
1.3.1 Info & Relationshipsrole="group" with a name associates the tiles as one set programmatically, matching the visual grouping.
1.4.1 Use of ColorThe set is conveyed by overlap, the ring, and the summary label — never colour.
1.4.11 Non-text ContrastThe ring is decorative separation, not a meaning-bearing boundary, so no 3:1 edge is relied upon.

Screen reader notes

  • In the summary pattern the group reads as its label, e.g. "Cardiology care team, 12 people, group", and the decorative tiles are skipped.
  • The "+N" tile is not announced; the total is already in the group's name.
  • The adjacent "view all" link is what a user follows to the named people.
  • Avoid leaving member initials unhidden — they read as letter strings ("E, R, J, M") and add only noise.
