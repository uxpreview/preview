Status dot

Beta

A small filled circle marking presence or state — open, busy, closed, live. In grayscale the dot cannot carry meaning on its own, so it is always paired with a text label; the dot is the quiet glance, the words are the message.

  • css/components/status-dot.css
  • Atom · no JavaScript

Overview

A status dot is the smallest possible state marker: a circle you read at a glance. Most systems lean on its colour — green for open, red for closed. This one can't, and shouldn't: the dot is decorative and the adjacent label is the status. The dot earns its place by drawing the eye to where that label is.

When to use

  • Presence or availability: a provider "Accepting patients", a location "Open until 8pm".
  • A short lifecycle state in a list or table, always beside its label.
  • A live indicator (with the pulse) for something happening now.

When not to use

  • Never alone. A bare dot communicates nothing in grayscale — pair it with text.
  • For a word or two of categorised status on an entity, prefer a Badge.
  • For a count on an icon, use the Badge count overlay, not a dot.

Anatomy

One circular span, marked decorative, sitting immediately before its label inside a small cluster so the two read as a unit.

dot + label Open now
PartClassRole
Dot.wire-status-dotThe circle. Decorative; carries the fill treatment and optional pulse.
Label— (text)The visible status word(s) beside the dot. The accessible meaning lives here.
Wrapper.u-clusterKeeps the dot and label together on one line (not part of this component).

Types & variants

Fill treatment and size compose. Because hue is off the table, the treatments map to emphasis — not to specific states. Bind a treatment to a meaning consistently within a screen, and let the label name the state.

VariantSelectorUse
Strong.wire-status-dotSolid, full-strength fill. The active or primary state.
Subtle.wire-status-dot--subtleMuted fill. An in-between or lower-emphasis state.
Outline.wire-status-dot--outlineHollow ring. The inactive, closed, or empty state.
Pulse.wire-status-dot--pulseA fading halo for a live state. Stills under reduced-motion.
Small / Large.wire-status-dot--sm / --lg0.5rem and 0.875rem; default is 0.625rem.
Sizes — sm · md · lg
Small Medium Large

Composability

A leaf that sits beside text. It composes into rows and headings rather than holding anything itself.

  • In a list or table — a dot + label in a status column.
  • On a heading — a "Live now" marker beside a section title.
  • With Wait time — a presence dot ahead of a current wait figure.
  • Not on an avatar corner — for presence pinned to an avatar, use the Badge anchor pattern with a small dot, not a free-floating one.

Do / Don't

Do

Open now

Always name the state in words. The dot is a glance cue; the label is the truth.

Don't

Bare dots with no labels say nothing in grayscale, and nothing to a screen reader.

Content guidelines

  • Label every dot. One or two plain words: "Open", "Closed", "Live now".
  • Keep the treatment-to-meaning mapping consistent within a screen.
  • Reserve the pulse for genuinely live, time-sensitive states.
  • Don't encode more than about three states with dots; past that, use text or a Badge.

Related

  • Badge — for a categorised status word, or a count overlay.
  • Wait time — pairs a presence dot with a current wait figure.
  • Avatar — for presence pinned to a person, via the Badge anchor.
  • All components — the full library.

References

Citations supporting this component, each labeled by evidence tier. Items marked unverified — pending review are drafts and must not be presented as established fact.

  • Standard — W3C WAI, WCAG 2.2 SC 1.4.1 Use of Color: status is never carried by colour (or a colour dot) alone; the text label conveys it. Unverified — drafted, pending review. w3.org
  • Standard — W3C WAI, WCAG 2.2 SC 4.1.3 Status Messages: a status that changes at runtime is announced via an aria-live region. Unverified — drafted, pending review. w3.org
  • Standard — W3C WAI, WCAG 2.2 SC 2.3.3 Animation from Interactions: the pulse is disabled under prefers-reduced-motion. Unverified — drafted, pending review. w3.org
  • Convention — Laws of UX (Jon Yablonski), Von Restorff (Isolation) Effect: a small high-contrast mark draws the eye to the status it sits beside. Unverified — pending review. lawsofux.com
  • Convention — Gestalt principle of proximity: the dot is read as belonging to the label immediately beside it. Unverified — pending review.

Measurements & structure

Like Badge's dot, the diameter is fixed component geometry in rem via the local --status-dot-size custom property, not a spacing token.

PropertyValue
Size — sm / md / lg0.5rem · 0.625rem · 0.875rem (--status-dot-size).
Shape--radius-pill on a square box; display: inline-block, vertical-align: middle.
Outlineinset box-shadow ring at --border-md in --color-border-strong; no fill.
PulseAn ::after halo scaling out and fading over --duration-slow, looping; animation: none under prefers-reduced-motion.

States

No interactive states (no hover, focus, active, or disabled) — the dot is never the control. Its visual set is the fill treatments and sizes in Usage › Types & variants. The pulse is a continuous indicator, not an interaction state.

JavaScript

None. The dot is static markup with no data-wire-* hooks. When the status it represents changes at runtime, the application updates the label inside an aria-live="polite" region so the change is announced — the dot itself never announces.

Code

<!-- Dot is decorative; the label carries the meaning -->
<span class="u-cluster u-cluster--xs">
  <span class="wire-status-dot" aria-hidden="true"></span>Open now
</span>

<!-- Fill treatments -->
<span class="wire-status-dot wire-status-dot--subtle" aria-hidden="true"></span>
<span class="wire-status-dot wire-status-dot--outline" aria-hidden="true"></span>

<!-- Live pulse (stills under reduced-motion) -->
<span class="u-cluster u-cluster--xs">
  <span class="wire-status-dot wire-status-dot--pulse" aria-hidden="true"></span>Live now
</span>

<!-- Runtime change announced via the label, not the dot -->
<span class="u-cluster u-cluster--xs" aria-live="polite">
  <span class="wire-status-dot" aria-hidden="true"></span>Open now
</span>

ARIA & semantics

  • The dot is always decorative: aria-hidden="true", no role, not focusable.
  • The visible text label is the status and the accessible name; never let the dot be the only signal.
  • Don't substitute a colour name in hidden text ("green") — name the state ("Open"), which is what the label already does.
  • For runtime changes, wrap the dot + label in an aria-live="polite" region so the new state is announced once.

Keyboard interactions

None. The dot is not interactive and not in the tab order. If the status sits inside a link or button, that control owns the keyboard behaviour; the dot adds nothing.

WCAG 2.2 criteria mapped

SCHow Status dot satisfies it
1.4.1 Use of ColorThe state is in the text label, never the dot's appearance. Grayscale guarantees no colour-only meaning.
1.4.11 Non-text ContrastThe dot is decorative and not meaning-bearing, so no 3:1 graphic boundary is relied upon; the label carries the information.
2.3.3 Animation from InteractionsThe pulse is disabled under prefers-reduced-motion: reduce.
4.1.3 Status MessagesRuntime changes are announced through an aria-live region on the label, not the dot.

Screen reader notes

  • Only the label is announced, e.g. "Open until 8:00pm"; the decorative dot is skipped.
  • A bare, unlabelled dot is announced as nothing — which is exactly why it must never stand alone.
  • The pulse is purely visual and is not announced.
  • Inside an aria-live="polite" region, a change from "Closed" to "Open" is read once when it happens.
