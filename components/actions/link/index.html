Link

Beta

A styled text link for navigation — taking the user somewhere, not performing an action. The base <a> already carries the system underline; .wire-link adds explicit treatments for a call-to-action, a quiet inline link, and links that carry a leading or trailing icon.

  • css/components/link.css
  • Atom · no JavaScript

Overview

The dividing line is simple: a link navigates, a button acts. If it changes the address bar or moves the user to other content, it is a link. If it submits, toggles, opens a dialog, or mutates data, it is a Button. Getting this right is the single most important link decision — it sets the right role, keyboard behaviour, and user expectation.

When to use

  • Inline within prose, to send the reader to a related page.
  • As a standalone call-to-action ("View all results →") at the end of a section.
  • For a download or a cross-site destination, with an icon that signals the change.

When not to use

  • For an action — submitting, opening a Modal, toggling. Use a Button.
  • As a fake link (<a> with no href wired to JS). Use a real button.
  • For a primary page action that needs prominence — that is a button's job.

Anatomy

An anchor with the system underline, an optional icon child, and a real href.

PartClass / attributeRole
Link.wire-link on <a href>The navigable text. Underlined by default.
Icon.wire-link__iconAn optional 1em <svg>, decorative; placed before or after the label by DOM order.
New-window note.u-visually-hidden text"(opens in a new window)" announced for target="_blank" links.

Inverse.wire-link--inverseFor a dark / inverse surface; keeps contrast.

Do / Don't

Do

Read the full visiting-hours policy.

Write link text that describes the destination, so it makes sense out of context (in a screen-reader links list).

Don't

Visiting-hours policy. Click here.

Don't use "click here" or a bare URL. The text alone should tell the user where they will land.

Content guidelines

  • Front-load the meaningful words ("Download the intake form", not "To download the intake form, click here").
  • Keep link text unique on a page when destinations differ; don't have three "Learn more" links to three different pages without context.
  • Signal a file type or new window in the text or an icon plus hidden text, not by colour.
  • Don't underline non-link text; reserve the underline for links.

Measurements & structure

PropertyValue
Underline offset0.18em, so descenders clear the line.
Underline weight--border-thin at rest → --border-md on hover/focus.
Icon.wire-link__icon at 1em, baseline-aligned (-0.125em).
Standaloneinline-flex with a --space-2xs gap so the icon tracks the label.

States

Hover thickens the underline; :focus-visible draws the global focus ring; :visited is left to the browser default (still distinguished by underline, not colour). The quiet variant adds an underline on hover and focus so it is never colour-only.

JavaScript

None. Link is a native anchor with no data-wire-* hooks.

Code

<!-- Inline link -->
<a class="wire-link" href="/visit-summary">visit summary</a>

<!-- Quiet link (dense UI) -->
<a class="wire-link wire-link--quiet" href="/billing">billing FAQ</a>

<!-- Standalone call-to-action with trailing arrow -->
<a class="wire-link wire-link--standalone" href="/specialties">
  Explore all specialties
  <span class="wire-link__icon" aria-hidden="true"><svg ... focusable="false">...</svg></span>
</a>

<!-- New window: announce the change with hidden text -->
<a class="wire-link" href="/rights.pdf" target="_blank" rel="noopener">
  Patient rights (PDF)
  <span class="wire-link__icon" aria-hidden="true"><svg ...>...</svg></span>
  <span class="u-visually-hidden"> (opens in a new window)</span>
</a>

ARIA & semantics

  • Use a real <a> with a valid href — that gives the link role and keyboard support for free.
  • Never wire a JS action onto an <a> without an href; if it acts rather than navigates, it is a Button.
  • Mark the icon decorative (aria-hidden, focusable="false"); it adds emphasis, not meaning.
  • For target="_blank", include visually-hidden "(opens in a new window)" and rel="noopener".

Keyboard interactions

KeyAction
TabMoves focus to the link; the global focus ring appears.
EnterFollows the link. (Links activate on Enter only — not Space, unlike buttons.)

WCAG 2.2 criteria mapped

SCHow Link satisfies it
2.4.4 Link Purpose (In Context)Guidance requires descriptive text; "click here" is called out as a Don't.
1.4.1 Use of ColorLinks are underlined, so they are not distinguished by colour alone — and the system is grayscale regardless.
2.4.7 Focus VisibleInherits the global focus ring; the inline box has a small radius so the ring is tidy.
1.1.1 Non-text ContentDecorative icons are hidden; a meaningful new-window cue is given as text.

Screen reader notes

  • A link reads its text and role, e.g. "visit summary, link".
  • Users often pull up a list of links out of context — which is why the text must stand alone.
  • The new-window cue is announced after the label: "Patient rights, PDF, opens in a new window, link".
