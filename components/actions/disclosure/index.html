Disclosure

Beta

A single show/hide for one chunk of secondary content. Built on the native <details>/<summary> elements, so the expanded state, keyboard support, and browser find-in-page work with no JavaScript. For a coordinated set of panels, use Accordion.

  • css/components/disclosure.css
  • Molecule · native, no JavaScript

Overview

Progressive disclosure keeps a page calm by hiding detail until it is asked for. A disclosure is the smallest form of that: one trigger, one panel, fully independent of any other. Because it is native, it never traps content from search — the browser can still find text inside a closed panel.

When to use

  • A single "Show more" / "Details" reveal beneath a summary.
  • An optional advanced section in a form that most users can skip.
  • One stand-alone FAQ item that isn't part of a set.

When not to use

  • For a group of related panels — use Accordion, which coordinates them.
  • For switching between peer views — that is Tabs.
  • For an overlay or transient panel — that is a Menu, Modal, or Drawer.
  • To hide content essential to every user; disclosure is for secondary detail.

Anatomy

A <details> wrapping a <summary> trigger and the revealed body. The chevron is ours; the open/closed state is the browser's.

PartClass / elementRole
Root.wire-disclosure on <details>Holds the open state natively; open attribute = expanded.
Summary.wire-disclosure__summary on <summary>The always-visible trigger. Focusable; ≥44px tall.
Icon.wire-disclosure__iconA chevron that rotates 180° when open. Decorative.
Body.wire-disclosure__bodyThe revealed content.

Do / Don't

Do

Use a native <details> so the panel toggles, takes focus, and is findable in-page for free.

Don't

Don't rebuild a disclosure from a <div> + JS when the native element already does it accessibly.

States

Closed and open are the native <details> states (the open attribute). Hover and :focus-visible apply to the summary; focus uses the global ring. The chevron reflects state by rotation — never by colour.

JavaScript

None — and that is the point. The native element handles toggling, state, keyboard, and find-in-page. No data-wire-* hook. (Animating the open/close height would require script and is intentionally omitted.)

Code

<details class="wire-disclosure">
  <summary class="wire-disclosure__summary">
    What should I bring to my appointment?
    <span class="wire-disclosure__icon" aria-hidden="true"><svg ... focusable="false">...</svg></span>
  </summary>
  <div class="wire-disclosure__body">
    <p>Bring a photo ID, your insurance card, and a list of medications.</p>
  </div>
</details>

<!-- Bordered, open by default -->
<details class="wire-disclosure wire-disclosure--bordered" open>
  <summary class="wire-disclosure__summary">Title <span class="wire-disclosure__icon" aria-hidden="true">...</span></summary>
  <div class="wire-disclosure__body">...</div>
</details>

ARIA & semantics

  • Native <details>/<summary> already expose the disclosure role and expanded state — no ARIA to add.
  • Keep the chevron decorative (aria-hidden, focusable="false"); the summary text is the accessible name.
  • Put a clear, self-describing label in the summary so its purpose is obvious when collapsed.
  • Don't override the element's behaviour with conflicting ARIA (e.g. a manual aria-expanded that can desync).

Keyboard interactions

KeyAction
TabMoves focus to the summary; the global focus ring appears.
Enter / SpaceToggles the panel open or closed.

WCAG 2.2 criteria mapped

SCHow Disclosure satisfies it
4.1.2 Name, Role, ValueRole and expanded state come from the native element; the summary is the name.
2.1.1 KeyboardEnter/Space toggle natively; the summary is in the tab order.
1.4.1 Use of ColorOpen/closed is shown by the chevron and revealed content, not hue.
2.5.8 Target Size (Min)The summary is ≥44px tall.

Screen reader notes

  • The summary reads as a disclosure/button with its expanded state, e.g. "What should I bring, collapsed".
  • Toggling announces the new state; the revealed body then reads in normal flow.
  • Because content is real DOM (not removed when closed), find-in-page can still locate it and expand it.
