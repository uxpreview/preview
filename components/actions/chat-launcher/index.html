Chat launcher

Beta

A floating action button that opens a chat, virtual-assistant, or help experience — usually pinned to the bottom corner of the screen. It is a launcher only: opening the actual conversation surface (a Modal, Drawer, or app panel) is the application's job.

  • css/components/chat-launcher.css
  • Molecule · no JavaScript

Live demo

The labelled launcher and the compact, round variant. Shown inline here; in production add --fixed to pin it to the bottom-end of the viewport.

Overview

A persistent, low-friction way to reach live help without leaving the page. In healthcare that might be a nurse line, scheduling help, or a billing question; the launcher stays put as the patient reads, then opens the conversation on demand. Because it floats over content, it must be modest in size and never cover the actions beneath it.

When to use

  • To offer live chat / virtual assistant help across a section or whole site.
  • As a persistent "Need help?" entry that follows the user down a long page.

When not to use

  • For a one-off in-page action — that is a Button.
  • For an emergency or crisis line — that needs a always-visible, unmissable treatment; see Emergency line.
  • When it would cover essential controls on small screens with no way to dismiss it.

Anatomy

PartClass / attributeRole
Launcher.wire-chat-launcher on <button>The floating action control.
Icon.wire-chat-launcher__iconA speech-bubble glyph. Decorative.
Label.wire-chat-launcher__label"Chat with us" — omitted in the compact variant (use aria-label).

Types & variants

VariantSelectorUse
Labelled.wire-chat-launcherIcon + label — clearest; preferred where space allows.
Compact.wire-chat-launcher--compactRound, icon-only; needs an aria-label.
Fixed.wire-chat-launcher--fixedPins to the bottom-end of the viewport (the production placement).

Do / Don't

Do

Chat with us

Label it plainly and keep it clear of footer and back-to-top controls.

Don't

Don't use the chat launcher for an emergency line, and don't let it permanently cover content on small screens.

References

Citations supporting this component, each labeled by evidence tier. Items marked unverified — pending review are drafts and must not be presented as established fact.

  • Standard — W3C WAI, WCAG 2.2 SC 4.1.2 Name, Role, Value: the launcher is a button with a text name and (when it opens a dialog) aria-haspopup/aria-expanded. w3.org
  • Standard — W3C WAI, WCAG 2.2 SC 2.5.8 Target Size (Minimum): the launcher clears the 24px floor and is comfortable for touch. w3.org

Measurements & structure

PropertyValue
Heightmin-block-size: 3.25rem — a large, easy target.
Compact3.25rem square circle, icon centred.

Layer--z-sticky so dialogs/toasts can still cover it.

States

Rest, hover (darker fill), and :focus-visible (global ring). When the launcher opens a dialog, the app sets aria-expanded="true" while it is open. There is no disabled state — a help entry is either present or not rendered.

JavaScript

None ships with the component — it has no data-wire-* hook. The application wires the click to open its chat surface and keeps aria-expanded in sync. If it opens a Modal or Drawer, that component handles its own focus management.

Code

<!-- Labelled, pinned bottom-end (production) -->
<button type="button" class="wire-chat-launcher wire-chat-launcher--fixed"
        aria-haspopup="dialog" aria-expanded="false">
  <span class="wire-chat-launcher__icon" aria-hidden="true"><svg ... focusable="false">...</svg></span>
  <span class="wire-chat-launcher__label">Chat with us</span>
</button>

<!-- Compact, icon-only — needs an aria-label -->
<button type="button" class="wire-chat-launcher wire-chat-launcher--compact wire-chat-launcher--fixed"
        aria-haspopup="dialog" aria-expanded="false" aria-label="Chat with us">
  <span class="wire-chat-launcher__icon" aria-hidden="true"><svg ...>...</svg></span>
</button>

ARIA & semantics

  • Use a real <button> with a clear name — "Chat with us", not just an icon.
  • The compact (icon-only) variant must carry an aria-label; the icon stays decorative.
  • Add aria-haspopup="dialog" and keep aria-expanded in sync with the open conversation surface.
  • When the chat opens in a Modal, move focus into it and return focus to the launcher on close (the Modal handles this).

Keyboard interactions

KeyAction
TabReaches the launcher; the global focus ring appears. Keep it in a sensible tab position even when fixed.
Enter / SpaceOpens the chat surface.

SCHow Chat launcher satisfies it

Screen reader notes

  • The launcher reads "Chat with us, button"; with a dialog popup it adds "has pop-up dialog, collapsed".
  • The compact variant relies entirely on its aria-label for the name.
  • Because it floats, confirm its position in the reading/tab order makes sense and that it never traps or hides focus.
