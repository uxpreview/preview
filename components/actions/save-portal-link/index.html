Save / portal link

Beta

Two related healthcare affordances in one small family. Save for later is a toggle that bookmarks a provider, location, or article so a patient can return to it. Patient-portal entry is a prominent link into the logged-in portal. Both share one pill control; the save toggle carries a persistent state that a plain Button does not.

  • css/components/save-portal-link.css
  • Molecule · no JavaScript

Live demo

A save toggle in both states, and the portal-entry link. The save control is a real <button> with aria-pressed; the portal link is an <a>.

An honest snapshot for this beta page. The save toggle's pressed state and the portal link are static markup; persistence (remembering what's saved) and the actual portal auth are the application's job. A screen-reader pass on the toggle wording is pending. Methodology lives in Validation.

Overview

Choosing care takes time, and patients compare across visits and devices. A save control lets them set a provider or location aside without committing; a portal link gives a single, recognisable door into their account. Both are small but high-traffic, so they read clearly and meet a comfortable target size for older hands.

When to use

  • Save — on a provider card, location, or article a patient may want to revisit.
  • Portal — in the header or a prominent banner, as the entry to the logged-in experience.

When not to use

  • For a generic primary action — use a Button.
  • For a one-off icon control with no persistent state — use an Icon button.
  • For favouriting many items in a dense grid where labels won't fit — an Icon button toggle may suit better.

Do / Don't

Do

Saved

Flip the label and fill the icon when saved, and carry the state on aria-pressed. The change is visible and announced.

Don't

Save

Don't show only a colour change for "saved", and don't leave off aria-pressed — the state would be invisible to assistive tech and in grayscale.

Measurements & structure

PropertyValue
Heightmin-block-size: 2.75rem — ≥44px target.
Layoutinline-flex, icon + label with a --space-2xs gap.
Icon1.25em; filled when saved, outline when not.
ShapePill (--radius-pill) with a 1px border.

States

Rest, hover (background wash), :focus-visible (global ring), and saved ([aria-pressed="true"]: firmer border + subtle fill + "Saved"). The portal variant has its own filled rest and hover. Disabled is rare here; if needed, use the native disabled attribute on the button.

JavaScript

None ships with the component. The application toggles aria-pressed and the label/icon, and persists what is saved. No data-wire-* hook.

Code

<!-- Save toggle (off). Flip aria-pressed + label + icon fill on save. -->
<button type="button" class="wire-save-link" aria-pressed="false">
  <span class="wire-save-link__icon" aria-hidden="true"><svg ... focusable="false">...</svg></span>
  <span class="wire-save-link__label">Save</span>
</button>

<!-- Saved (on) -->
<button type="button" class="wire-save-link" aria-pressed="true">
  <span class="wire-save-link__icon" aria-hidden="true"><svg fill="currentColor" ...>...</svg></span>
  <span class="wire-save-link__label">Saved</span>
</button>

<!-- Portal entry: an anchor, not a button -->
<a class="wire-save-link wire-save-link--portal" href="/portal">
  <span class="wire-save-link__icon" aria-hidden="true"><svg ...>...</svg></span>
  <span class="wire-save-link__label">Log in to patient portal</span>
</a>

ARIA & semantics

  • Save is a toggle <button> with aria-pressed — it acts (changes state), so it must be a button, not a link.
  • Portal is an <a href> — it navigates, so it must be a link, not a button.
  • The label is real text inside the control, so the accessible name is always present; the icon is decorative.
  • Keep the label honest with the state: "Save" when off, "Saved" when on (don't make the user infer it from the icon alone).

Keyboard interactions

KeyAction
TabFocuses the control; global focus ring appears.
Enter / SpaceToggles save (button) — links follow on Enter only.

Screen reader notes

  • The save toggle reads "Save, toggle button, not pressed" → "Saved, toggle button, pressed" after activation.
  • The portal link reads "Log in to patient portal, link".
  • Because the label text changes with state, the change is announced even without relying on aria-pressed alone.
