Field

Stable

A labeled input: a stack of label, control, and optional hint or error. .wire-field is the wrapper; the control inside is a text .wire-input, a .wire-textarea, or a .wire-select.

  • css/components/form.css
  • Since v1.0

Live demo

A field is a label bound to a control, with optional hint text below. Everything is keyboard- and screen-reader-ready out of the box.

Default field
As it appears on your insurance card.

Anatomy

Four parts, stacked. The control is required; label is required; hint and error are optional and mutually exclusive at a glance.

PartClassRole
Label.wire-field__labelNames the field; bound to the control with for/id.
Control.wire-input · .wire-textarea · .wire-selectThe input the user types into or chooses from.
Hint.wire-field__hintQuiet helper text under the control.
Error.wire-field__errorReplaces the hint when the field is invalid.

When to use

  • Use .wire-input for short single-line text; .wire-textarea for multi-line free text.
  • Use .wire-select for a small, fixed list of mutually exclusive options.
  • Reach for a different control when the input isn't free text — Search, Choice (checkbox/radio/toggle), and Form layout are documented separately.

Avoid a .wire-select for more than ~10 options (use search or autocomplete), and never add a bare required asterisk — use the .wire-field__label-required modifier so the marker stays consistent.

In context

Fields stack in a form. Keep one column for scannability; pair related short fields in a row only when they're clearly connected.

A short intake

Do / Don't

Do

  • Bind every label to its control with for/id.
  • Put format help in the hint, before the user types.
  • Keep forms to a single column.

Don't

  • Use placeholder text as the label — it vanishes on focus.
  • Mark required fields with a bare * instead of the modifier.
  • Pack format rules into a tooltip the user can't see while typing.

Labels

  • Every control has a visible <label> bound with for/id — never a placeholder standing in for a label (WCAG 3.3.2).
  • Clicking the label focuses the control, enlarging the effective target.
  • Hint text is announced as part of the field when linked with aria-describedby.

Required & optional

  • Mark required fields with .wire-field__label-required, which adds a consistent visual marker; pair it with required on the control so the requirement is programmatic, not just visual.
  • If most fields are required, mark the few optional ones instead — whichever is the shorter list.

Errors

  • On invalid, add .is-invalid to the control and link the message with aria-describedby so a screen reader reads the error with the field.
  • Error copy says what happened and how to fix it — "Enter a date in MM / DD / YYYY format", not "Invalid input".
  • Color is never the only signal: the heavier border and the text message carry the state too (WCAG 1.4.1).

Keyboard & input

KeyAction
TabMoves to the control in source order.
Type / arrowsEdit text, or open and move through a select's options.
EscCloses an open select without changing the value.

Use a real type (email, tel, search) and an autocomplete token so mobile keyboards and autofill work. Controls keep a 44px-tall target.
