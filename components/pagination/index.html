Pagination

Stable

A control for moving through a long, ordered set one page at a time — previous and next, numbered pages, and an ellipsis where the middle is collapsed. The current page is shown, not linked.

  • css/components/pagination.css
  • Since v1.0

Live demo

Previous and next flank a numbered list. First and last stay reachable; the gap between collapses to an ellipsis. Page 2 is current.

Anatomy

A labeled landmark holds optional previous/next controls around an ordered list of page links. The current page and the ellipsis are non-interactive.

PartClassRole
Landmark.wire-paginationA <nav> with aria-label="Pagination".
Prev / Next.wire-pagination__navStep controls; aria-disabled="true" at the ends.
List.wire-pagination__listThe page numbers.
Page link.wire-pagination__linkA link to a specific page.
Current.wire-pagination__currentThe current page as a filled, non-link span with aria-current="page".
Ellipsis.wire-pagination__ellipsisA decorative gap for skipped pages.

When to use

  • Use pagination to split a long, ordered set into pages users move through — search results, a provider directory, an article archive.
  • Reach for it when the total count or a specific page matters, or users want to jump to the first or last page.
  • For an open, continuous feed, a "load more" or infinite scroll fits better — pagination is for bounded, indexable sets.
  • It's not for steps in a process (use a stepper) or for hierarchy (use a Breadcrumb).

Do / Don't

Do

  • Show the current page as a non-link with aria-current="page".
  • Disable previous on the first page and next on the last.
  • Keep first and last reachable; collapse the middle with an ellipsis.

Don't

  • Link the current page to itself.
  • Remove prev/next when disabled — dim them so position stays stable.
  • Use pagination for unbounded, continuously loading content.

Structure

  • A <nav> landmark holds optional prev/next controls flanking an unordered list of page links.
  • Page links and the current page are 40px minimum squares (min-inline-size and min-block-size: 2.5rem) with a pill radius.
  • The current page is a filled <span>, not a link; the ellipsis is a non-interactive span.
  • Disabled prev/next carry aria-disabled="true" (40% opacity, pointer-events: none) — visible but inert, so the row doesn't reflow at the ends.

Snippet

<nav class="wire-pagination" aria-label="Pagination">
  <a class="wire-pagination__nav" href="?page=1" aria-disabled="true">← Previous</a>
  <ul class="wire-pagination__list" role="list">
    <li class="wire-pagination__item">
      <span class="wire-pagination__current" aria-current="page">1</span>
    </li>
    <li class="wire-pagination__item"><a class="wire-pagination__link" href="?page=2">2</a></li>
    <li class="wire-pagination__item"><span class="wire-pagination__ellipsis" aria-hidden="true">…</span></li>
    <li class="wire-pagination__item"><a class="wire-pagination__link" href="?page=9">9</a></li>
  </ul>
  <a class="wire-pagination__nav" href="?page=2">Next →</a>
</nav>

Landmark & labeling

  • Wrap the control in <nav aria-label="Pagination"> so it's a distinct landmark.
  • Mark the current page with aria-current="page" on a non-link <span>.
  • Give prev/next real text ("Previous", "Next") — an arrow glyph alone isn't an accessible name.

Disabled & decorative

  • At the ends, set aria-disabled="true" on prev/next rather than removing them — they stay announced and the layout stays put.
  • The ellipsis is decorative (aria-hidden="true"): it marks skipped pages, not a control.

Keyboard & target size

KeyAction
TabMoves across prev, page links, and next in order.
EnterActivates the focused page link or step control.

The current-page span and the ellipsis aren't focusable. Every control clears a 40px target, comfortably past the WCAG 2.5.8 minimum.
