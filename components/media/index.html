Media

Stable

Greybox image and video placeholders that hold an aspect ratio while a wireframe stands in for real imagery, plus a gallery grid that arranges several together. Decorative by default — production swaps in real images with alt text.

  • css/components/media.css
  • Since v1.0

Live demo

A greybox at a fixed ratio. The data-ratio attribute prints a corner label; an optional .wire-media__label names what the image will be.

Default (16:9) with a label

When to use

  • Use a placeholder to reserve image space at a chosen ratio while the layout is still grey — it keeps the page from reflowing when real images land.
  • Reach for a gallery when three or more placeholders sit together; a single image is just one .wire-media.
  • Use the video variant where a player will go; use mosaic for a feature wall, even columns for a uniform grid.
  • These are wireframe stand-ins, not final imagery — production replaces each with a real <img> or <video>.

Do / Don't

Do

  • Pick the ratio the real image will use, so nothing reflows later.
  • Name the intended image with a label during wireframing.
  • Replace placeholders with real images carrying meaningful alt text.

Don't

  • Ship a greybox as a finished image.
  • Put meaningful content only inside a decorative placeholder.
  • Mix wildly different ratios in one even-column gallery.

Structure

  • The ratio is a custom property (--media-ratio) driving aspect-ratio; each modifier just re-sets it (16:9, 4:3, 1:1, 3:4, 2:3, 21:9).
  • The greybox texture is two diagonal gradient lines forming an X — a recognizable "image goes here" cue.
  • The corner label is generated from attr(data-ratio); the video variant swaps the texture for a dark fill and a CSS play triangle.

Snippet

<!-- single placeholder -->
<div class="wire-media wire-media--16-9" aria-hidden="true" data-ratio="16:9">
  <span class="wire-media__label">Hero image</span>
</div>

<!-- gallery -->
<div class="wire-gallery wire-gallery--3">
  <div class="wire-media wire-media--square" aria-hidden="true" data-ratio="1:1"></div>
  …
</div>

Decorative by default

  • Placeholders carry aria-hidden="true" — they're greyboxes with no information, so they're hidden from assistive tech (WCAG 1.1.1).
  • The label and corner ratio are inside the hidden element; they orient sighted reviewers during wireframing, not end users.

In production

  • Each placeholder is replaced by a real <img> with meaningful alt text (or empty alt="" when the image is purely decorative).
  • A video placeholder becomes a real player with captions and accessible controls.
  • Keep the same ratio so the swap doesn't shift the layout.

Contrast

  • The greybox and its X lines are non-text decoration, exempt from text-contrast rules; they remain visibly distinct in both themes.
  • The corner label sits on an opaque chip so it stays legible over the texture.
