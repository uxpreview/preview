Text size

Stable

A user-controlled text-size scale — small, medium, large — that persists the choice and announces it to assistive tech. It sits in a utility bar or settings panel as an in-page aid for long-form and older-audience content; it complements browser zoom rather than replacing it.

  • css/components/text-size-control.css
  • js/wire.js

Live demo

A working control. Pick a size — small, medium, or large — and the paragraph below rescales with the rest of the page. The choice is saved, so it persists if you reload or move to another page in the system.

Text-size control — small / medium / large
Text

After surgery, take it slow. Walk a little more each day, keep your follow-up appointments, and call the care team if anything feels wrong. This paragraph scales with the control above — the same way every block of body text on the page does — so the choice carries through the whole reading experience, not just one element.

Anatomy

A container holds a short label and three size buttons. The container carries the JS hook; each button carries its own value. Buttons reuse a size modifier to render the "A" glyph at its matching scale.

PartClassRole
Container.wire-text-size + data-wire-text-sizeThe pill group; carries the JS hook and a role="group" with an accessible name.
Label.wire-text-size__labelA short visible "Text" label inside the group.
Small button.wire-text-size__btn--sm + data-wire-text-size-value="sm"Selects the small scale (0.9×).
Medium button.wire-text-size__btn--md + data-wire-text-size-value="md"Selects the default medium scale (1×).
Large button.wire-text-size__btn--lg + data-wire-text-size-value="lg"Selects the large scale (1.15×).

When to use

  • Offer it in a utility bar or settings panel for long-form or older-audience content — care instructions, plan-your-visit pages, policy text.
  • Treat it as a complement to browser zoom, not a replacement. Native zoom and OS text settings stay available; this control is an extra, on-page affordance for people who never change those.
  • Don't make it the only way to get readable text. Default body type must already meet size minimums on its own.
  • Pair it with the other senior-friendly affordances — it sits naturally beside a Help bar or click-to-call.

Do / Don't

Do

  • Persist the choice so it carries across pages and reloads.
  • Announce the change to assistive tech via the polite live region.
  • Give the group an accessible name so its purpose is clear out of context.

Don't

  • Disable or override native browser zoom.
  • Make this the only path to legible text — default type must already pass.
  • Ship a default body size that depends on the user bumping it up.

Behavior

Wired by js/wire.js. On load it finds every [data-wire-text-size] group and reads the saved value from localStorage under the key wire-text-scale, defaulting to md. Selecting a button does three things:

  • Sets a scale on the root. It writes a multiplier to the --wire-text-scale custom property on :root; the stylesheet applies it via html { font-size: calc(100% * var(--wire-text-scale)); }, so every rem-based size rescales together.
  • Persists the choice. It saves the chosen value to localStorage, so the size carries across reloads and other pages in the system.
  • Announces the change. After the initial load, it writes "Text size: small / medium / large" into a polite live region so screen readers hear the change. The first apply on page load is silent — only later, user-initiated changes are announced.

Selecting a button also updates aria-pressed across every text-size group on the page, keeping the active size in sync if more than one control is present. The scale values are sm = 0.9×, md = 1×, lg = 1.15×.

</div>

Semantics

  • The buttons form a group: the container carries role="group" with an aria-label ("Text size"), so the set has an accessible name out of context.
  • Each button is a real <button> and carries aria-pressed. The JS keeps exactly one button pressed at a time, so the active size is conveyed to assistive tech, not signalled by appearance alone.
  • The buttons exceed the minimum target size — each is at least 2.75rem (44px) square, satisfying WCAG 2.2 SC 2.5.8 Target Size (Minimum) with margin, and visible focus is preserved.

Resize text

  • This control directly supports WCAG 2.2 SC 1.4.4 Resize Text, which requires text to be resizable up to 200% without loss of content or function. The control is an on-page aid toward that goal.
  • It is not a substitute for meeting 1.4.4. The page itself must still satisfy the criterion through browser zoom and reflow — the control adds a discoverable affordance for users who don't reach for native zoom.
  • Because the scale is applied as a root font-size multiplier, all rem-based type scales together; the layout must hold at the largest setting without clipping or overlap.

Announcement

  • On change, the JS writes "Text size: small", "Text size: medium", or "Text size: large" into a visually-hidden polite live region (role="status", aria-live="polite") so screen-reader users hear the result of their action.
  • The announcement fires only for user-initiated changes. The initial apply on page load — restoring a saved choice — is intentionally silent, so a screen reader isn't interrupted on every page entry.
