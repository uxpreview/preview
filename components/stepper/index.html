Stepper

Beta

A progress indicator for a flow with a handful of discrete, ordered steps — a booking, an application, an onboarding sequence. It shows where the user is, what they've finished, and what's left, so a multi-screen task feels bounded instead of endless. It marks progress; it does not navigate.

  • css/components/stepper.css
  • Molecule · no JavaScript

Live demo

A four-step booking flow on its third step. Horizontal from tablet up, stacked vertical on phones — resize to see it switch. Each step's state is spoken to screen readers, not just shown.

  1. Completed: Choose provider Dr. Okonkwo
  2. Completed: Pick a time Tue, June 9, 10:30 AM
  3. Confirm Review & submit
  4. Done

Overview

A stepper gives a multi-step task a shape. Seeing "step 3 of 4" tells the user the end is near and that two answers are already banked — which lowers the chance they abandon halfway. It works only when the flow is genuinely linear and short; a long or branching flow makes the straight line lie.

When to use

  • A linear flow of roughly 3–7 discrete steps — booking, application, checkout, onboarding.
  • A task where progress and remaining effort are worth showing up front.
  • A sequence where each step is a distinct, nameable stage.

When not to use

  • A single step — there is no progress to show.
  • A branching or non-linear flow — a straight row of steps misrepresents it.
  • More than about seven steps — group them, or the indicator becomes its own wall.
  • As navigation — the stepper reflects progress; it is not a set of links to jump between stages.

Anatomy

An ordered list of items; each carries a marker, a label, and optional meta. The state class drives the look, and parallel text or aria-current carries the same state to assistive tech.

PartClassRole
Root.wire-stepper (on <ol>)The ordered list; its order conveys the sequence. Give it an aria-label.
Item.wire-stepper__item (on <li>)One step. Carries .is-complete, .is-current, or neither (upcoming).
Marker.wire-stepper__markerThe numbered / ✓ disc, drawn by CSS. Decorative — aria-hidden="true".
Label.wire-stepper__labelThe step name; holds the visually-hidden state text.
Meta.wire-stepper__label-metaOptional secondary line — the chosen value, a sub-label.

Behaviors

States

Each item is in exactly one of three states. The class sets the visual; a parallel cue sets the accessible meaning so it never rides on color alone.

StateClass / attributeCue to assistive tech
Complete.is-completeVisually-hidden "Completed: " inside the label.
Current.is-current + aria-current="step"aria-current="step" is announced as the current step.
Upcoming(no class)Neither cue — read as a future step in the ordered list.

Responsive

Horizontal with connector lines from 48em up; below that it stacks vertically with a short connector between markers. Nothing is hidden — the same steps, re-flowed. For the connector spec and tokens, see the Specs tab.

In context

  • hospital-appointment.html — heads the booking confirmation with a four-step progress row.
  • hospital-booking.html — the pick-a-time step in the same flow.
  • hospital-trial.html — the eligibility sequence on a clinical-trial page.
  • university-program.html — an apply-sequence stepper in higher-ed.

Do / Don't

Do

Show a short linear path: Choose → Time → Confirm → Done.

Four named stages, one current — the user knows exactly where they are.

Don't

Force a branchy flow into a straight row.

If steps can be skipped or reordered, the line lies — use sections or a wizard instead.

Content guidelines

  • Name each step with a verb or a noun the user recognizes: "Pick a time", not "Step 2".
  • Use the meta line to echo the choice made — "Tue, June 9, 10:30 AM" — so completed steps stay informative.
  • Keep labels to one or two words where you can; the row gets crowded fast.
  • Match the count to reality. Don't pad a two-step task to look more thorough.

States

What each state means lives in Usage › Behaviors.

StateTriggerWhat changes
Upcoming(default)Open disc, muted ring and label; the marker shows its number.
Current.is-currentAccent-filled disc, full-weight label, leading connector accented.
Complete.is-completeAccent-filled disc with ✓; the connector to the next step turns accent.

JavaScript

None. The stepper is markup and CSS. Advance it server-side or by swapping the state classes (.is-complete / .is-current) and the matching aria-current and visually-hidden text as the user moves between steps.

Code

<ol class="wire-stepper" aria-label="Booking progress">
  <li class="wire-stepper__item is-complete">
    <span class="wire-stepper__marker" aria-hidden="true"></span>
    <span class="wire-stepper__label"><span class="u-visually-hidden">Completed: </span>Choose provider</span>
  </li>
  <li class="wire-stepper__item is-current" aria-current="step">
    <span class="wire-stepper__marker" aria-hidden="true"></span>
    <span class="wire-stepper__label">Confirm</span>
  </li>
  <li class="wire-stepper__item">
    <span class="wire-stepper__marker" aria-hidden="true"></span>
    <span class="wire-stepper__label">Done</span>
  </li>
</ol>

ARIA & semantics

  • The list is an <ol>, so order — the heart of a stepper — is conveyed semantically. Give it an aria-label naming the flow ("Booking progress").
  • The current step carries aria-current="step", which screen readers announce as the current step.
  • Completed steps carry a visually-hidden "Completed: " inside the label, so the state isn't conveyed by the marker color alone (SC 1.4.1).
  • The marker disc is decorative — aria-hidden="true" — because its number / ✓ is CSS-generated and would otherwise read as noise.

WCAG 2.2 criteria mapped

SCHow Stepper satisfies it
1.3.1 Info and RelationshipsThe ordered list conveys sequence; the current step is marked with aria-current.
1.4.1 Use of ColorComplete / current / upcoming are conveyed by text and aria-current, not the accent fill alone.
2.4.6 Headings and LabelsEach step is named with a meaningful label, not just a number.
1.4.10 ReflowThe row stacks to a vertical list under 48em with no loss of steps or meaning.

Screen reader notes

  • A reader hears the list's label, then each item in order: "Completed: Choose provider", "Confirm, current step", "Done".
  • The marker's number / ✓ is hidden from the reader on purpose; the label text and aria-current carry the meaning.
  • When the user advances, update the state class, the aria-current, and the visually-hidden text together so the spoken state stays in sync with the visual.
  • Pending: a live NVDA + VoiceOver pass of the new state text is not yet logged — this is why Stepper is beta.
