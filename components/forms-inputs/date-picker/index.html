Date picker

Beta

Lets someone enter a date — a date of birth, an appointment day. It's the native <input type="date"> in a Field: you can type the date or open the browser's own calendar. That native calendar is accessible, localized, and works on touch out of the box, so Preview leans on it rather than shipping a custom grid.

  • Native — no JavaScript

Overview

A custom date picker is one of the easiest components to get wrong — keyboard traps, missing announcements, broken localization. The native input avoids all of it and is familiar to users, so it's the default here. Help people by bounding the range with min/max and showing the expected format in the hint. For a known date far in the past (a date of birth), typing is faster than paging a calendar back decades — the native input supports both, so don't take typing away.

When to use

  • Any single calendar date (appointment, start date, date of birth).
  • Where a real, validated date value is needed and the native calendar is acceptable.

When not to use

  • For a month/year only, or a relative choice ("today", "next available") — use a Select or buttons.
  • For a date range as one control — use two bounded date inputs in a Fieldset ("from" / "to").
  • When a bespoke calendar UI is mandated — that's a custom build beyond this native-first component (flag it).

Measurements & behaviour

PropertyValue
BaseUses .wire-input (≥2.75rem tall, shared border/focus/disabled/invalid).

Code

<div class="wire-field">
  <label class="wire-field__label" for="appt">Preferred appointment date</label>
  <input type="date" class="wire-input" id="appt"
         min="2026-06-15" max="2026-12-31" aria-describedby="appt-hint">
  <p class="wire-field__hint" id="appt-hint">Choose any weekday in the next six months.</p>
</div>

ARIA & semantics

  • Use the native <input type="date"> with a real <label for> — no ARIA needed for name, role, or value.
  • State the expected format in the hint; the native control accepts typed dates and the calendar both.
  • Set aria-invalid="true" with .is-invalid and a .wire-field__error when validation fails.
  • Don't replace it with a div-based calendar; if a bespoke calendar is truly required, treat it as a separate custom build.

  • Native support and calendar-popup announcements vary by browser; the typed segments are the consistent path.
