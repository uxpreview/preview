Slider

Beta

Lets someone pick a number from a continuous range by dragging — a budget, a distance, a comfort level. It's the native <input type="range"> styled grayscale, with a visible value readout. Best when the rough position matters more than the exact figure; for a precise number, pair it with — or use — a Number stepper.

  • css/components/form.css
  • data-wire-slider

Live demo

Drag the handle or use the arrow keys; the readout updates live. A native range in a Field, with the value echoed in an <output>.

Testing status

A new beta component. The control is a native range input, so its role, value, and keyboard model come from the platform; the JavaScript only mirrors the value into a visible readout. A formal cross-screen-reader pass and a check of the custom thumb/track at high zoom are pending. Methodology lives in Validation.

Overview

A slider trades precision for feel: it's quick and tangible for setting an approximate value, but hard to land on an exact one, fiddly on touch, and invisible to anyone who needs to type a figure. Use it where "about here" is good enough and the range is bounded and meaningful. When the exact number matters — a dose, an age, a dollar amount someone knows — give them a field to type into instead.

When to use

  • Approximate, bounded values where dragging feels natural (budget ceiling, distance radius, brightness).
  • Filters where users explore a range and see results respond.

When not to use

  • For an exact figure the user already knows — use a Number stepper or Field.
  • For an unbounded value (no sensible min/max) — sliders need both ends.
  • For a yes/no or small option set — use a Toggle or Choice.

Code

<div class="wire-field">
  <label class="wire-field__label" for="budget">Monthly premium budget</label>
  <div class="wire-slider" data-wire-slider data-wire-slider-suffix=" / mo">
    <div class="wire-slider__row">
      <input type="range" class="wire-slider__range" id="budget"
             min="0" max="600" step="25" value="250"
             aria-describedby="budget-hint">
      <output class="wire-slider__output" for="budget">250 / mo</output>
    </div>
  </div>
  <p class="wire-field__hint" id="budget-hint">Set the most you'd like to pay each month.</p>
</div>

ARIA & semantics

  • Use the native range input — it is role="slider" with aria-valuemin/max/now maintained by the browser.
  • Give it a real <label for>; the <output> is supplementary, not the label.
  • If the number needs units to make sense ("250 dollars", not "250"), set aria-valuetext so AT speaks the unit too.
  • State the min and max in the label or hint so the scale is clear without sight of the track.

Screen reader notes

  • Reads as "Monthly premium budget, slider, 250" and re-announces the value as it changes.
  • Without aria-valuetext, only the bare number is spoken — add it when units matter.
