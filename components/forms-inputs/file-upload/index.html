File upload

Beta

Lets someone attach a file — a referral letter, an insurance card photo, a transcript. It's the native <input type="file"> hidden behind a styled "Choose file" button, with the chosen filename shown and announced beside it. The native picker means platform, antivirus, and cloud integrations all keep working.

  • css/components/form.css
  • data-wire-file

Live demo

Activate the button to open the system file picker; the chosen filename appears — and is announced — beside it.

default
No file chosen

PDF, JPG, or PNG, up to 10 MB.

Testing status

A new beta component. The control is a native file input (hidden with .u-visually-hidden but still focusable), so the picker, role, and value are the platform's; the JavaScript only mirrors the filename into a live status. Pending: confirming the visually-hidden input takes focus and shows the button's focus ring across browsers, and that the filename is announced once (not twice). Drag-and-drop is intentionally out of scope. Methodology lives in Validation.

Native picker Keyboard-operable Color-independent SR audit pending

See the validation log →

Overview

Keep file upload boring and native. The platform's own picker is familiar, works with cloud drives and assistive tech, and carries the user's antivirus and permissions — a custom drop zone throws that away and is hard to make keyboard-accessible. Here the only enhancement is cosmetic: a button you can style and a filename you can see and hear. State the accepted types and size limit up front, in the hint, so a rejected file isn't the first feedback.

When to use

  • Attaching documents or images to a form (referrals, ID photos, transcripts).
  • Anywhere you need a real file from the user's device or cloud storage.

When not to use

  • For a value the user types or selects — that's a Field or Select.
  • As a drag-only drop zone with no button — that excludes keyboard users.
  • For capturing a photo inline — use the platform capture attributes on the native input rather than a bespoke camera UI.

Code

<div class="wire-field">
  <label class="wire-field__label" id="ref-label">Upload your referral letter</label>
  <div class="wire-file" data-wire-file>
    <label class="wire-file__button">
      <input type="file" class="wire-file__input u-visually-hidden"
             aria-labelledby="ref-label" accept=".pdf,.jpg,.png">
      <span>Choose file</span>
    </label>
    <span class="wire-file__status" data-wire-file-status aria-live="polite">No file chosen</span>
  </div>
  <p class="wire-field__hint">PDF, JPG, or PNG, up to 10 MB.</p>
</div>

ARIA & semantics

  • Hide the input with .u-visually-hidden, never display:none — it must stay focusable and operable.
  • Label the input with the field label (aria-labelledby or a wrapping <label>); the icon is aria-hidden.
  • Draw the visible focus ring on the button via :focus-within so keyboard focus is obvious.
  • Keep the status aria-live="polite" so the chosen filename is announced.

Keyboard interactions

Screen reader notes

  • Reads as "Upload your referral letter, button" (the styled label) and opens the OS picker on activation.
  • After selection, the polite status announces the filename; the input also exposes the value natively.
