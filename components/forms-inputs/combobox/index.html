Combobox

Beta

A text input that filters a list as you type — for picking one value from a list too long to scroll comfortably, like a specialty, a medication, or a clinic. It follows the ARIA combobox pattern: type to narrow the options, arrow through them, Enter to choose. For a short, stable set, a plain Select is simpler and sturdier.

  • css/components/form.css
  • data-wire-combobox

Live demo

Type to filter (try "ol" or "card"), or use the chevron. Arrow keys move the highlight, Enter selects, Escape closes.

Testing status

A new beta component, and the most custom in this set — it implements the ARIA combobox pattern (role=combobox + listbox, aria-activedescendant) by hand, so it carries the most risk. The keyboard model and filtering are wired; what's pending is a thorough screen-reader pass across NVDA/JAWS/VoiceOver (activedescendant announcements vary), plus confirming results-count feedback. Where a short, fixed list will do, prefer Select. Methodology lives in Validation.

Overview

A combobox earns its complexity only when a list is long enough that filtering genuinely helps — dozens of specialties, hundreds of medications. Below that, it's a liability: a custom widget where a native Select would be more robust. When you do reach for it, keep the input a real text field (so typing always works), make every option reachable by keyboard, and never hide the chosen value behind colour alone. This is the one component here that isn't native, so it's held to the ARIA Authoring Practices to the letter.

When to use

  • Choosing one value from a long list where typing to filter saves real time (specialty, drug, city).
  • Lists that are awkward to scroll in a plain select.

When not to use

  • For a short, stable set (under ~10–15 items) — use a Select.
  • For choosing several values — that's a multi-select pattern, out of scope here.
  • For free text with no fixed list — that's a plain Field.

Code

<div class="wire-field">
  <label class="wire-field__label" for="spec">Find a specialty</label>
  <div class="wire-combobox" data-wire-combobox>
    <input type="text" class="wire-input wire-combobox__input" id="spec"
           role="combobox" aria-expanded="false" aria-controls="spec-list"
           aria-autocomplete="list" autocomplete="off">
    <button type="button" class="wire-combobox__toggle" tabindex="-1" aria-label="Show specialties">
      <svg class="wire-combobox__toggle-icon" viewBox="0 0 24 24" aria-hidden="true"><path d="m6 9 6 6 6-6"/></svg>
    </button>
    <ul class="wire-combobox__listbox" id="spec-list" role="listbox" aria-label="Specialties" hidden>
      <li class="wire-combobox__option" role="option" id="spec-1" aria-selected="false">Cardiology</li>
      <!-- …more options… -->
      <li class="wire-combobox__empty" hidden>No matching specialties</li>
    </ul>
  </div>
</div>

ARIA & semantics

  • The input is role="combobox" with aria-expanded, aria-controls → the listbox, and aria-autocomplete="list"; it keeps a real <label for>.
  • The popup is role="listbox" of role="option" items; the highlighted option is referenced by aria-activedescendant (focus never leaves the input).
  • Selection sets aria-selected="true" on the chosen option; the active highlight uses a tint and an edge bar, not colour alone.
  • The toggle is tabindex="-1" with an aria-label — a pointer convenience, not a second tab stop.

Keyboard interactions

KeyAction
↓ / ↑Open the list (if closed) and move the active option.
EnterSelect the active option and close.
EscClose the list, keeping focus in the input.
Home / EndMove to the first / last matching option.
TypeFilter the options; the list reopens.

Screen reader notes

  • Reads as "Find a specialty, combobox"; as you arrow, the active option ("Cardiology, 1 of 10") is announced via activedescendant.
  • activedescendant support varies across NVDA/JAWS/VoiceOver — this is the main reason the page is beta pending a cross-AT audit.
