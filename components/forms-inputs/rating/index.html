Rating

Beta

A star score, in two modes. Display shows an existing rating — a provider's average on a directory card — read-only. Input lets someone leave a rating, as a star-styled radio group. Filled versus outline stars carry the value by shape, so it survives grayscale.

  • css/components/form.css
  • data-wire-rating (input mode)

Top: a read-only display rating with its value and review count. Bottom: an input rating — hover, focus, or arrow through the stars to set a score.

Testing status

A new beta component. Display mode is static markup with an aria-label carrying the score — robust and JS-free. Input mode is a native radio group (the source of truth) with JavaScript only painting the hover/focus preview, so keyboard and AT fall back to plain radios. Pending: confirming each radio's per-star label is announced clearly and the preview never desyncs from the checked value. Half-star display is out of scope. Methodology lives in Validation.

Display: no JS Input: native radios Shape-based (grayscale) SR audit pending

See the validation log →

Overview

Pick the mode by intent. Showing an existing score? Use display — it's read-only, so it must never look or behave like something you can click. Collecting a score? Use input, and treat it as exactly what it is: a single-choice question, built on radios. Either way, the number must never live in colour alone — filled and outline stars differ in shape, and display mode pairs the stars with the figure in text so the score is unambiguous to everyone.

When to use

  • Display: average ratings on directory or product cards (a provider's 4.0 from 128 reviews).
  • Input: letting a patient rate a visit, a course, or an article.

When not to use

  • For a longer satisfaction scale or labelled options — use a Choice (radio) group with text.
  • For yes/no helpfulness ("Was this useful?") — two buttons are clearer than stars.
  • Don't make a display rating focusable or clickable; it isn't a control.

Code

<!-- Display (read-only) -->
<span class="wire-rating" role="img" aria-label="Rated 4 out of 5 stars, 128 reviews">
  <svg class="wire-rating__star wire-rating__star--filled" viewBox="0 0 24 24" aria-hidden="true"><path d="…star…"/></svg>
  <!-- four filled, one outline -->
  <svg class="wire-rating__star" viewBox="0 0 24 24" aria-hidden="true"><path d="…star…"/></svg>
  <span class="wire-rating__value" aria-hidden="true">4.0 (128)</span>
</span>

<!-- Input (radio group) -->
<fieldset class="wire-fieldset">
  <legend class="wire-fieldset__legend">Rate your visit</legend>
  <div class="wire-rating wire-rating--input" data-wire-rating>
    <label class="wire-rating__option">
      <input type="radio" name="rate-visit" value="1" class="u-visually-hidden">
      <svg class="wire-rating__star" viewBox="0 0 24 24" aria-hidden="true"><path d="…star…"/></svg>
      <span class="u-visually-hidden">1 star</span>
    </label>
    <!-- …repeat for 2–5 stars… -->
  </div>
</fieldset>

ARIA & semantics

  • Display: wrap the stars in role="img" with an aria-label that states the score ("Rated 4 out of 5 stars, 128 reviews"); the star SVGs are aria-hidden.
  • Input: use a radio per star inside a Fieldset with a legend; each radio carries a visually-hidden text label ("3 stars").
  • Never use colour as the only cue — the shape (filled/outline) and, in display, the text value carry the score.
  • The JS preview is decorative; if it fails, the radios still work.

Keyboard interactions (input mode)

KeyAction
TabMoves into the rating group (to the checked star, or the first).
← / →, ↑ / ↓Move between stars and select (native radio behaviour).

Display mode is not focusable — it's an image, not a control.

SCHow Rating satisfies it

Screen reader notes

  • Display reads as a single image: "Rated 4 out of 5 stars, 128 reviews" — not five separate stars.
  • Input reads as a radio group: "Rate your visit, 3 stars, radio button, 3 of 5".
