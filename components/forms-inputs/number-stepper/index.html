Number stepper

Beta

A number input with −/+ buttons for small, exact whole-number values — how many refills, how many dependents, how many guests. The native <input type="number"> stays the value; the buttons are a faster path for mouse, touch, and anyone who finds typing fiddly. Use it for small counts with clear limits, not open-ended figures.

  • css/components/form.css
  • data-wire-number-stepper

Live demo

Use the buttons, or focus the field and press the arrow keys. The buttons disable at the limits.

default

Choose between 0 and 6 refills.

Testing status

A new beta component. The field is a native number input, so its role (spinbutton), value, and arrow-key behaviour are the platform's; the JavaScript only wires the −/+ buttons and disables them at the bounds. Pending: confirming the button-driven value change is announced consistently across screen readers (the arrow-key path on the field is the reliable one). Methodology lives in Validation.

Overview

A stepper shines for small whole numbers with obvious limits, where tapping up or down beats summoning a keyboard. It stops being the right tool as the range grows: nobody wants to tap "+" forty times. Keep the typed field fully usable — never lock people into the buttons — and set min, max, and step so the value can't go out of bounds.

When to use

  • Small counts with clear bounds (refills, dependents, tickets, guests).
  • Touch contexts where ±1 taps are quicker than typing.

When not to use

  • For large or wide-ranging numbers — use a plain Field (numeric) instead.
  • For an approximate value where the feel matters more than the figure — use a Slider.
  • For a value with no sensible step (free decimals, currency to the cent) — a typed field is clearer.

Code

<div class="wire-field">
  <label class="wire-field__label" for="refills">Number of refills</label>
  <div class="wire-number-stepper" data-wire-number-stepper>
    <button type="button" class="wire-number-stepper__btn wire-number-stepper__btn--dec" aria-label="Decrease number of refills"><span aria-hidden="true">−</span></button>
    <input type="number" class="wire-number-stepper__field" id="refills"
           value="2" min="0" max="6" step="1" inputmode="numeric">
    <button type="button" class="wire-number-stepper__btn wire-number-stepper__btn--inc" aria-label="Increase number of refills"><span aria-hidden="true">+</span></button>
  </div>
</div>

ARIA & semantics

  • Keep the native number input as the value carrier — it is a spinbutton with a real <label for>.
  • Give each button a clear aria-label ("Decrease number of refills"); the −/+ glyphs are aria-hidden decoration.
  • Buttons are type="button" so they never submit the surrounding form.
  • Never remove the typed field; it's the reliable path for keyboard and screen-reader users.

Keyboard interactions

KeyAction
↑ / ↓ (on field)Step the value up / down by step.
Type a numberEnter the value directly.
Tab, Enter/Space (on buttons)Focus and activate the −/+ buttons.

SCHow Number stepper satisfies it
4.1.2 Name, Role, ValueNative spinbutton field; buttons named via aria-label.

1.4.1 Use of ColorThe disabled bound is shown by the disabled attribute and dimming, not colour alone.

Screen reader notes

  • The field reads as "Number of refills, spin button, 2"; arrow keys announce each new value.
  • The buttons read by their labels ("Decrease number of refills, button"); when one disables at a bound, it's announced as dimmed/unavailable.
