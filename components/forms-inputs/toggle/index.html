Toggle

Beta

A switch for a single on/off setting that takes effect immediately — text alerts on or off, large-text mode on or off. It's a native checkbox with role="switch", styled as a sliding switch. Use it for instant binary settings, not for form choices you submit later.

  • css/components/form.css
  • Since v1.0 · no JavaScript

Live demo

A switch off, on, and disabled. Each is a real checkbox with role="switch"; the label is part of the control, so clicking the text toggles it.

Overview

A toggle is for a setting that flips now — the change applies the instant it's switched, with no Save button. That immediacy is the whole distinction: if the value is only collected and submitted with the rest of a form, it should be a checkbox, not a switch. Pair every toggle with a clear, static label that names the setting; the on/off state lives in the switch, not in changing label text.

When to use

  • An immediate on/off preference (notifications, larger text, high-contrast).
  • Settings screens where each change takes effect at once.

When not to use

  • For a value submitted with a form later — use a checkbox.
  • For choosing among several options — use radios or a Select.
  • For an action that happens once — use a Button.

Anatomy

A <label> wrapping the switch input and its text, so the label and control are inseparable.

PartClass / elementRole
Wrapper.wire-toggle on <label>Lays out switch + text; the whole thing is the target (≥44px).
Switch.wire-toggle__input on <input type="checkbox" role="switch">The control; a sliding knob shows on/off.
Label text<span>The static name of the setting.

States

StateAttributeAppearance
Off(unchecked)Knob left, neutral track.

Do / Don't

Do

Keep a static label naming the setting; the switch shows on/off.

Don't

Don't make the label the state ("On"/"Off") — the user can't tell what it controls, and it duplicates the switch.

Measurements & structure

PropertyValue
Track2.5rem × 1.5rem pill.
Knob1.25rem circle, 0.125rem inset; slides 1rem on check.

Base.wire-toggle__input is an appearance:none checkbox; the knob is its ::after.

JavaScript

None. The native checkbox handles state and keyboard; styling is pure CSS off :checked. No data-wire-* hook. The application listens for change to apply the setting immediately.

Code

<label class="wire-toggle">
  <input type="checkbox" class="wire-toggle__input" role="switch">
  <span>Text message reminders</span>
</label>

<!-- On by default -->
<label class="wire-toggle">
  <input type="checkbox" class="wire-toggle__input" role="switch" checked>
  <span>Larger text</span>
</label>

<!-- Disabled -->
<label class="wire-toggle">
  <input type="checkbox" class="wire-toggle__input" role="switch" disabled>
  <span>Email reminders (unavailable)</span>
</label>

ARIA & semantics

  • Use a native checkbox with role="switch"; its checked state is the on/off value — no extra ARIA needed.
  • The wrapping <label> is the accessible name; keep it static and descriptive.
  • Don't swap the label text for "On"/"Off" — the switch already conveys state, and a changing name is confusing.
  • If a toggle has a heading-level group, wrap related toggles in a Fieldset with a legend.

Keyboard interactions

KeyAction
TabMoves focus to the switch; the global focus ring appears.
SpaceToggles on/off (native checkbox behaviour).

WCAG 2.2 criteria mapped

SCHow Toggle satisfies it
4.1.2 Name, Role, ValueNative checkbox + role="switch" exposes name, role, and on/off value.
2.1.1 KeyboardFocusable and toggled with Space.
1.4.1 Use of ColorOn/off is shown by knob position, not colour alone.
2.5.8 Target Size (Min)The label+switch row is ≥44px tall.

Screen reader notes

  • Reads as "Text message reminders, switch, off" → "on" after activation.
  • Some older AT may announce a role="switch" as a checkbox ("checked/not checked"); the meaning is equivalent.
