Search

Stable

A text input tuned for finding things: a leading magnifier and the search input type. Use it to filter or query a known set of records, paired with a results list or a set of filters.

  • css/components/form.css
  • Since v1.0

Live demo

A full-width search input with a leading magnifier. The icon is decorative; the input carries the accessible name.

Search field
⌕

Anatomy

A relatively-positioned wrapper holds a leading icon and the input; the input is padded to clear the icon.

PartClassRole
Wrapper.wire-searchPositioning context for the icon.
Icon.wire-search__iconDecorative leading magnifier (aria-hidden).
Input.wire-input.wire-search__inputA type="search" input, padded for the icon.

When to use

  • Use Search to find or filter a known set — providers, conditions, articles — usually above a results list or beside a filter set.
  • Use a plain Field for free-text data you're collecting, not querying.
  • Use a select or radio when the choices are short and fixed — search is for open-ended lookup.

Give the magnifier a partner: search should lead somewhere — results, suggestions, or a filtered view — never sit decoratively with nothing behind it.

In context

Search shows up two ways: a prominent page-level query, and a compact filter inside a sidebar or toolbar.

Do / Don't

Do

  • Give the input a name — a visible label or aria-label.
  • Use type="search" so the browser offers a clear control.
  • Place it directly above or beside what it searches.

Don't

  • Rely on the placeholder as the only label.
  • Use Search for collecting free-text input — that's a Field.
  • Show a search box that leads to nothing.

Search inherits the input states: rest, hover, focus-visible, and disabled. With a value, the browser's native clear control appears for type="search".

Labeling

  • The input needs an accessible name: a visible <label> where one fits, or an aria-label="Search" when the magnifier alone implies the purpose.
  • The placeholder is a hint, not a label — it disappears on input and isn't reliably announced.
  • The magnifier is decorative (aria-hidden="true"); it never carries the name.

Semantics

  • Use type="search" — it signals intent to assistive tech and gives a native clear control.
  • When search submits a query, wrap it in a <form role="search"> so it's exposed as a search landmark.

Keyboard

KeyAction
EnterSubmits the query (when inside a form).
EscClears the field (native type="search" behavior).
TabMoves to and from the input in source order.

The input keeps a 44px-tall target and a visible focus ring.
