Banner

Beta

A full-width strip that carries context affecting the whole page — a scheduled outage, a policy change, a campus closure. It stays put while the user keeps working, which is what separates it from a toast that vanishes and a modal that blocks. Reserve it for messages worth a persistent line at the top of the view.

  • css/components/banner.css
  • Molecule · dismiss needs js/wire.js

Overview

A banner speaks to the whole page, not to one action. It is the pattern for "you should know this while you're here": an outage, a closure, a policy that changes what the user can expect. It persists so the context stays available, and it doesn't block, so the user keeps working around it.

When to use

  • Site- or section-level context the user should carry — planned downtime, a closure, a policy update.
  • A standing notice that should remain visible rather than flash by.
  • A message with at most one supporting action ("Read the changes"), not a task in itself.

When not to use

  • To confirm something the user just did — that is a toast.
  • For a decision they must make before continuing — that is a modal.
  • For inline editorial emphasis inside a page's flow — that is a callout.
  • To hold critical information in a dismissable banner — once closed, the user can't get it back.

Anatomy

An optional icon, a content column of title + body, an optional actions cluster, and an optional dismiss. It stacks on phones and goes inline from tablet up.

PartClassRole
Root.wire-bannerThe strip; sets the surface and border via the variant.
Icon.wire-banner__iconA bordered glyph (i / !). Decorative — keep aria-hidden="true".
Content.wire-banner__contentStacks title and body.
Title.wire-banner__titleThe headline of the notice.
Body.wire-banner__bodyOne or two sentences of detail.
Actions.wire-banner__actionsAn optional cluster — usually a single secondary button or link.
Close.wire-banner__closeThe × control; a 44px target that removes the banner (needs JS).

Types & variants

Four variants, all grayscale. Emphasis comes from surface and border weight, never hue — the alert variant reads as serious because its border is heavier, not because it is red.

VariantSelectorUse for
Default.wire-bannerStandard informational notice on a muted surface.
Subtle.wire-banner--subtleA quieter notice with a hairline border — environment notes, low-stakes context.
Inverse.wire-banner--inverseA dark strip for high contrast against a light page region.
Alert.wire-banner--alertHigh emphasis — a heavier border and bolder title for outages and urgent notices.

Behaviors

States

A banner is simply present or dismissed. There is no open/closed animation and no persisted state — dismissing removes the element for the session. For the visual treatment and tokens, see the Specs tab.

Interactions

Mouse

Clicking the × removes the banner from the page. Any action button behaves like a normal link or button.

Keyboard

The dismiss and any action are normal controls in the tab order; Enter or Space activates them. The full contract is in the Accessibility tab.

In context

  • hospital-medication.html — a safety banner heads the medication detail with dosing cautions.
  • hospital-appointment.html — a banner carries booking-window context across the flow.

Do / Don't

Do

Carry standing context: "Portal maintenance Sunday 2–4 AM."

Something true for the whole visit, left visible while the user works.

Don't

Confirm a one-off in a banner: "Your changes were saved."

A momentary acknowledgement belongs in a toast, not a standing strip.

Related

  • Toast — to confirm a one-off action without leaving a standing line.
  • Modal — when the user must act before continuing.
  • Callout — for editorial emphasis inside the body, not page-level status.
  • All components — the full library.

References

Citations supporting this component, each labeled by evidence tier. Items marked unverified — pending review are drafts and must not be presented as established fact.

  • Standard — W3C WAI, WCAG 2.2 SC 1.3.1 Info and Relationships: the title / body / actions structure is conveyed through markup, so the relationship survives without the visual layout. Unverified — pending review. w3.org

JavaScript

The banner renders with no JavaScript. Only dismissal needs js/wire.js — a non-dismissable banner is fully static.

  • .wire-banner__close — initBanner wires every close button to remove its closest .wire-banner. No data attribute is required.
  • Dismissal is not persisted — a reload brings the banner back. Persist it yourself if it should stay closed.

Code

<div class="wire-banner wire-banner--alert" role="alert">
  <span class="wire-banner__icon" aria-hidden="true">!</span>
  <div class="wire-banner__content">
    <div class="wire-banner__title">Portal maintenance — Sunday 2–4 AM CT</div>
    <div class="wire-banner__body">Booking will be unavailable.</div>
  </div>
  <div class="wire-banner__actions">
    <a class="wire-button wire-button--sm wire-button--secondary" href="#">Details</a>
  </div>
  <button class="wire-banner__close" aria-label="Dismiss banner">×</button>
</div>
<script src="js/wire.js" defer></script>

ARIA & semantics

  • By default a banner is just text in the page flow — no role, no live region. It is read in source order like any other content.
  • For urgent content that appears or changes dynamically (an outage going live), add role="alert" so assistive tech announces it at once.
  • The icon is decorative and carries aria-hidden="true"; the meaning lives in the title and body text.
  • The dismiss button has an explicit aria-label="Dismiss banner".

Keyboard interactions

KeyAction
TabMoves to the action button (if any), then the dismiss — in source order.
Enter / SpaceActivates the focused action or dismiss (native).

WCAG 2.2 criteria mapped

SCHow Banner satisfies it
1.3.1 Info and RelationshipsTitle, body, and actions are real elements, so the structure survives without the visual strip.
1.4.1 Use of ColorThe alert variant signals urgency with border weight and bolder type, not hue. Grayscale throughout.
2.2.2 Pause, Stop, HideA persistent notice can be dismissed, so it never permanently occupies the viewport.
2.5.8 Target Size (Minimum) · AAThe × dismiss target is 44 × 44 — past the 24px floor.
4.1.2 Name, Role, ValueThe dismiss is a labelled button; an urgent banner exposes its role via role="alert".

Screen reader notes

  • A static banner is announced when the reader reaches it. Add role="alert" only when the banner appears or changes after load and the user needs to hear it immediately.
  • Don't put must-know information solely in a dismissable banner — once closed it leaves the accessibility tree along with everything else.
  • Pending: a live NVDA + VoiceOver pass — including the role="alert" path — is not yet logged. This is why Banner is beta.
