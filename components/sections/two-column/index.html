Two-column

Stable

A balanced two-column layout pairing a headline and supporting content — reversible, with optional vertical alignment and width-ratio variants.

  • css/components/two-column.css
  • 6 variants

Live demo

Text and media in a 50/50 split. A text column carries an eyebrow, title, supporting prose, and an optional actions row; the media column is a decorative placeholder here.

Text + media (default)
Our approach

Therapy that starts the day you arrive.

Most rehab hospitals wait until you are stable. We start the moment it is safe — because the brain rewires fastest in the first weeks after injury.

Read the research

Anatomy

A grid container holds two children: a text column and a second slot (media or another text column). The text column stacks an eyebrow, title, body, and an optional actions row. Variants change the ratio, vertical alignment, and order — the parts stay constant.

PartClassRole
Container.wire-twocol + variantTwo-column grid; collapses to one column on narrow viewports.
Text column.wire-twocol__textFlex column wrapping the eyebrow, title, body, and actions.
Eyebrow.wire-twocol__eyebrowShort uppercase kicker above the title.
Title.wire-twocol__titleThe column headline (a heading element in real use).
Body.wire-twocol__bodySupporting paragraph, muted.
Actions.wire-twocol__actionsOptional row of buttons or links.
Media slot.wire-media + ratioThe second column when pairing text with an image.

When to use

  • Pair a short headline with an image — a program photo, facility shot, or portrait — when the picture is part of the message.
  • Set two related blocks of prose side by side when neither should dominate the page.
  • Reach for a ratio variant when one column needs more room than the other — a wide image beside a narrow caption, or a lead paragraph beside a tight aside.
  • For a full-width emphatic page opening, use a Hero instead; the two-column block is a mid-page content pairing, not the first block.

Variants

One container, six modifiers: a reverse for column order, two for vertical alignment, and three for width ratio. All collapse to a single stacked column below the tablet breakpoint.

Reversed — media leads on desktop, text still first in the DOM

Top-aligned — columns of unequal height align at the top

40 / 60 ratio — narrow text beside a wider media slot
Patient story

A stroke survivor's recovery, from first steps to her first 5K.

33 / 66 ratio — a tight aside beside a lead paragraph
The model

Research at the bedside.

Traditional medical research lives in universities — far from the patients it is meant to help. We built something different: scientists work in labs embedded inside the hospital, sharing elevators with the clinicians delivering care. The result is a faster, tighter feedback loop between discovery and treatment.

Do / Don't

Do

  • Keep the text column first in the DOM so it reads before the image, even when --reverse flips the visual order.
  • Pick a ratio that matches the content weight — a wide image earns --40-60; a tight caption earns --33-66.
  • Use a real heading element for the title when the block introduces a section.

Don't

  • Reorder the columns with --reverse just to change reading order — it only flips the visual layout.
  • Stack more than two children in the grid; add a third column with a Feature grid instead.
  • Rely on left/right position alone to convey meaning; the columns stack on small screens.

Modifiers combine freely — a ratio with an alignment, or either with reverse. The base container is an even 50/50 split.

VariantClassEffect

Source order

  • The text column comes first in the DOM, so screen readers and keyboard users reach the message before the image — even when --reverse shows the media on the left.
  • --reverse is a CSS order change for visual layout only; it never alters the reading or focus order. Don't lean on it to fix reading order (WCAG 2.2 SC 1.3.2 Meaningful Sequence).
  • The media slot here is decorative scaffolding (aria-hidden="true"). A real image needs meaningful alt text, or empty alt if it's purely decorative (WCAG 2.2 SC 1.1.1).

Headings

  • The title is styled text, not a fixed heading level. When the block opens a section, mark the title up as the right heading for the page outline — usually an h2 or h3.
  • Keep the eyebrow as plain text; it labels the title and isn't a level in the document outline (WCAG 2.2 SC 1.3.1 Info and Relationships).

Responsive

  • Below the tablet breakpoint the grid collapses to one column and every variant stacks vertically, text column first — so position never carries meaning that's lost when columns stack.
  • Ratio and alignment modifiers only apply at wider viewports; the stacked layout is a single full-width column at every ratio.
  • Content reflows without horizontal scrolling and stays legible when zoomed to 200% (WCAG 2.2 SC 1.4.10 Reflow).
