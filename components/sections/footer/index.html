Footer

Stable

The page-closing block — brand, navigation columns, and a legal line — in four densities: compact for tools and sign-offs, standard for a four-column sitemap, expanded when a final call to action belongs at the bottom, and inverse for a dark, high-emphasis close.

  • css/components/footer.css
  • 4 variants

Anatomy

A footer stacks a brand block, optional link columns, an optional CTA row, and a bottom row with copyright and legal links — all inside a single wire-footer__inner column. Variants change which parts appear and how densely they stack; the parts themselves stay constant.

PartClassRole
Container.wire-footer + variantThe landmark block; takes one density modifier.
Inner.wire-footer__innerStacks the rows; sits inside a .u-container.
Brand.wire-footer__brand, __brand-mark, __taglineLogo lockup and a short positioning line.
Columns.wire-footer__columnsGrid of link groups — 2-up on mobile, 4-up from 48em.
Column title.wire-footer__col-titleLabel above each list — styled like a heading, not one.
Link list.wire-footer__list, __list-item, __linkA real ul/li of footer links.
CTA.wire-footer__cta, __cta-titleClosing call-to-action row — expanded variant only.
Bottom.wire-footer__bottom, __legalCopyright plus an inline list of legal links.

When to use

  • Use compact on tools, dashboards, and internal docs where the footer is a sign-off, not a sitemap.
  • Use standard (the default, no modifier) on marketing and content sites that need clear navigation by topic.
  • Use expanded when a final conversion moment belongs at the bottom of the page — an appointment, donation, or signup.
  • Use inverse when the page above has finished light and a dark close gives a strong, intentional bookend.

When not to use

  • Don't duplicate the top nav in the footer; the footer is for breadth (sitemap), not active wayfinding.
  • Don't place a primary CTA in a compact footer — reach for expanded.
  • Don't use inverse if the section directly above is already dark; the page loses its rhythm.

For a section opener rather than a page close, reach for a Page header or Hero instead.

One container, four densities. Compact collapses to a single row; expanded adds a CTA above the columns; inverse flips the surface dark and pairs with any of the others.

Do / Don't

Do

  • Use the real <footer> element so the block is a contentinfo landmark.
  • Group links by topic and give each column a clear, scannable title.
  • Keep legal links (privacy, accessibility, nondiscrimination) in the bottom row.

Don't

  • Mirror the entire top nav — the footer is breadth, not active wayfinding.
  • Promote a column title to a real heading; it labels the list, it isn't an outline level.
  • Stack two footers, or use inverse below an already-dark section.

Landmark

  • Render the footer as a real <footer> element so it exposes the contentinfo landmark; assistive tech can then jump straight to it.
  • Use one site-level footer per page. A <footer> nested inside an article or section is not contentinfo, so keep the page-closing footer at the top level of the document body.

Links & lists

  • Each column of links is a true <ul role="list"> of <li> items, so screen readers announce the list and its length.
  • The column title above is a heading-style label, not a true heading — it stays a <div class="wire-footer__col-title"> and never an h*, keeping the page outline clean.
  • Link text must read clearly out of context; vertical links carry a 1.75rem minimum block size plus padding so they meet the AA target size (WCAG 2.2 SC 2.5.8).

Contrast & focus

  • The inverse variant restyles link and column-title colors to a light ramp value so body and link text stay at or above 4.5:1 on the dark surface (WCAG 2.2 SC 1.4.3).
  • On the inverse surface the focus token flips to a light value, keeping the focus ring visible (WCAG 2.2 SC 2.4.7).
  • Legal links stay at a 12px minimum with an underline on hover — never hidden behind tiny, low-contrast type.
  • Hierarchy comes from scale, weight, and space — never color — so the footer reads the same in light and dark themes.
