Quote

Stable

A pull quote that surfaces a person's words — a patient, a clinician, a researcher — set in a serif display with their attribution. Three variants: default, large display, and an inline left-bar.

  • css/components/quote.css
  • Since v1.0

Live demo

A blockquote carries the words; a figcaption attributes them. The opening mark is a decorative serif glyph.

Default

Anatomy

PartClassRole
Figure.wire-quoteA <figure> wrapping the quotation and its caption.
Mark.wire-quote__markA decorative opening quotation glyph (aria-hidden); default variant only.
Text.wire-quote__textA <blockquote> with the words.
Attribution.wire-quote__attributionA <figcaption> holding name and role.
Name / Role.wire-quote__name · __roleWho said it, and in what capacity.

Variants

The structure is constant; the scale and framing change.

Large (display)
Inline (left-bar)

When to use

  • Use a pull quote to surface a person's words for rhythm and credibility — a testimonial, a researcher's framing.
  • Default for a standard quote; large for a display moment at the top of a story; inline for a short quotation inside running text.
  • It's for someone's words, not editorial emphasis (use a Callout) and not a statistic.
  • Keep it short — a pull quote is a highlight, not a transcript.

Do / Don't

Do

  • Attribute every quote with a name and role.
  • Keep it to a sentence or two.
  • Pair the words and source with figure / figcaption.

Don't

  • Invent or paraphrase an attribution.
  • Use a quote for non-quotation emphasis.
  • Stack several pull quotes in a row.

Type & scale

  • The quotation is a serif italic display; attribution is sans. The contrast in voice is the point.
  • Size scales with the variant: default --text-2xl → --text-3xl at 48em; large climbs to --text-5xl; inline sits at --text-lg → --text-xl.
  • text-wrap: balance keeps the display lines even; the figure caps at --container-md for a readable measure.

Tokens

PropertyToken
Quote text--font-serif, italic, --leading-snug
Mark--text-5xl, --color-text-subtle
Name--text-md, --weight-semibold
Role--text-sm, --color-text-muted
Inline bar--border-thick --color-border-strong
Measure--container-md

Snippet

<figure class="wire-quote">
  <div class="wire-quote__mark" aria-hidden="true">"</div>
  <blockquote class="wire-quote__text">They taught me how to expect more.</blockquote>
  <figcaption class="wire-quote__attribution">
    <span class="wire-quote__name">Maya Sundaresan</span>
    <span class="wire-quote__role">Patient, Stroke Recovery Program</span>
  </figcaption>
</figure>

Semantics

  • Use <figure> + <blockquote> + <figcaption>: the blockquote is the quotation, the figcaption its source. This binds the words to their attribution programmatically.
  • Don't wrap the attribution inside the blockquote — only the quoted words belong there.

Decorative mark

  • The large opening quotation glyph is decorative and marked aria-hidden="true" — the real quotation marks live in the punctuation a screen reader already reads.
  • It's the only variant-specific element; large and inline drop it.

Readability

  • The figure caps at a comfortable measure (--container-md) so a display quote never runs to an unreadable line length.
  • All text holds AA contrast in both themes; the role line uses muted text that still clears AA.
