Form layout

Stable

The scaffolding that assembles fields into a form: .wire-form stacks fields in one column, .wire-form__row places a few side by side, and .wire-fieldset groups related choices under a legend.

  • css/components/form.css
  • Since v1.0

Anatomy

PartClassRole
Form.wire-formStacks its children in one column with even spacing.
Row.wire-form__row.wire-form__row--2Places two short, related fields side by side (stacks on mobile).
Group.wire-fieldset + .wire-fieldset__legendGroups related choices under a named legend.
Actions.u-cluster of buttonsPrimary first, then secondary; at the end of the form.

When to use

  • Default to a single column — it's faster to complete and lowers errors.
  • Use a __row--2 only for fields that are short and clearly paired (first / last name, city / ZIP).
  • Wrap a set of radios or checkboxes in a .wire-fieldset so the group has a name.
  • End with one primary action and an optional secondary; never two primaries.

Do / Don't

Do

  • Keep one column as the default; pair fields only when they belong together.
  • Order fields the way a person would expect to answer them.
  • Put the primary action first in the cluster.

Don't

  • Lay out a long form in multiple columns — it's harder to scan and skips fields.
  • Group unrelated fields into a row just to save vertical space.
  • Offer two equally-weighted primary buttons.

Single column

  • A single column is the default for a reason: multi-column forms are scanned unpredictably and lead to skipped fields and more errors.
  • Reserve __row--2 for short, obviously-paired fields; everything else stays stacked.

Grouping & order

  • Group related controls in a <fieldset> with a <legend> so assistive tech announces the set's name.
  • Order fields the way a person reasons about them; the tab order follows source order, so keep them aligned.

Actions

  • End with one primary action; a secondary (Cancel) is optional and visually quieter.
  • The submit button is a real type="submit" so Enter submits the form, and it keeps the 44px target and focus ring.
  • Field-level labels, required marking, and errors are covered on the Field page.
