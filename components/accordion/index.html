Accordion

Stable

A vertically stacked set of disclosure items built on the native <details> element. Each item expands independently to reveal its content; an optional single-open mode keeps only one section open at a time.

  • css/components/accordion.css
  • Native <details> / <summary>

Live demo

Single-open mode (data-wire-accordion="single"): opening one item closes the others. Tab to a header and press Enter or Space to toggle it.

Overview

An accordion collapses a set of related sections so a user sees the headings first and expands only what they need. It trades immediate visibility for a shorter, scannable page. Reach for it when content is secondary, self-contained, and not everyone needs every part.

When to use

  • FAQs and other question-and-answer content.
  • Long supporting detail (policies, specs, fine print) that would otherwise crowd the page.
  • Sections a user reads selectively rather than top to bottom.
  • Dense mobile layouts where collapsing reduces scrolling.

When not to use

  • Do not hide content most users need. If it is essential, show it.
  • Do not use it to compare sections side by side; users cannot hold two collapsed panels in view. Reach for a table or columns.
  • Do not use it for a linear, ordered process. Use a stepper instead.
  • With only two short items, skip the accordion and show them outright.

Anatomy

A container holds a stack of items. Each item is a native <details> with a <summary> header (label plus the +/− indicator) and a body that is revealed when open.

container · item · summary · icon · body
Summary (the header label)

Body — the content revealed when the item is open.

Types & variants

Accordion has a single structural type (a stacked disclosure list) and no size or emphasis modifiers. The one meaningful variant is the open mode, set on the container.

Behaviors

States

Each item has two functional states: collapsed (default) and expanded (the native [open] attribute). The +/− icon indicates which: + collapsed, − expanded. Items begin collapsed unless you mark one open; starting collapsed gives a high-level overview first. For the visual treatment and tokens behind each state, see the Specs tab.

Interactions

Mouse

Clicking anywhere on the summary header toggles that item, not just the icon. In single-open mode, opening one item closes the others.

Keyboard

Tab moves focus between summary headers; Enter or Space toggles the focused item. This is native to <details>. For the full keyboard contract, see the Accessibility tab.

Composability

An accordion holds a list of items, each a question (the summary) paired with an answer (the body).

  • Summary — a short label, ideally a question or a clear noun phrase. Keep it to one line where possible.
  • Body — prose, lists, links, or a small table. It is capped at a 70ch measure for readability.
  • Open by default — add the open attribute to the one item most users want first.
  • Avoid nesting — do not place an accordion inside an accordion body; the disclosure layers become hard to track.

In context

A billing FAQ near the foot of a patient-resources page, collapsing six questions into a scannable list.

Billing & insurance FAQ
When will I receive my bill?

Statements are mailed within 30 days of your visit once insurance has processed the claim. You can also view balances anytime in the patient portal.

What if I don't have insurance?

Our financial counselors can help you apply for assistance or set up a payment plan. Call the billing office to get started.

Can I dispute a charge?

Yes. Submit a billing inquiry through the portal or by phone within 60 days, and a specialist will review your account.

Do / Don't

Do

How do I refill a prescription?

Request refills in the portal or call the pharmacy line.

One clear question per item, front-loaded with the keyword.

Don't

Information about prescriptions, refills, pharmacy hours, and more

Vague, multi-topic labels force users to open every item to find anything.

Don't bundle several topics behind one vague label, and never hide critical safety or cost information.

Content guidelines

Labels

  • Write summaries as the user's question ("When will I receive my bill?") or a clear noun phrase.
  • Front-load the keyword so a scan of collapsed headers is enough to navigate.
  • Keep one topic per item. If a body covers several ideas, split it.
  • Keep bodies scannable: short paragraphs, lists over walls of text.

Composition patterns

  • FAQ blocks — single-open, near the foot of a resources or service page.
  • Detail pages — collapsing specs, policies, or eligibility on a provider, program, or location page.
  • Mobile long-form — collapsing secondary sections to cut scroll length.
  • Plan-your-visit — grouping parking, check-in, and what-to-bring under scannable headers.

JavaScript

Multi-open is the default and ships zero JavaScript — it is the native <details> element. JavaScript is only needed for single-open mode.

  • Hook — data-wire-accordion="single" on the container. initAccordion in js/wire.js listens for the native toggle event and closes sibling items when one opens.
  • Include <script src="js/wire.js" defer></script> only when you use single-open mode (or other interactive components on the page).
  • The +/− swap and all visual states are CSS-only and work with or without script.

Code

<!-- Multi-open (native, no JS) -->
<div class="wire-accordion">
  <details class="wire-accordion__item" open>
    <summary class="wire-accordion__summary">
      <span>What should I bring?</span>
      <span class="wire-accordion__icon" aria-hidden="true"></span>
    </summary>
    <div class="wire-accordion__body">
      <p>Bring a photo ID and your insurance card.</p>
    </div>
  </details>
  <details class="wire-accordion__item">
    <summary class="wire-accordion__summary">
      <span>How early should I arrive?</span>
      <span class="wire-accordion__icon" aria-hidden="true"></span>
    </summary>
    <div class="wire-accordion__body"><p>Fifteen minutes before your visit.</p></div>
  </details>
</div>

<!-- Single-open (closes siblings; needs js/wire.js) -->
<div class="wire-accordion" data-wire-accordion="single">
  <!-- same item markup -->
</div>

ARIA & semantics

  • Built on native <details> / <summary>. The browser exposes the summary as a button and reports its expanded or collapsed state for free.
  • Do not add role, aria-expanded, or tabindex by hand. The native element manages all of it; manual ARIA would conflict.
  • The +/− icon is decorative and carries aria-hidden="true"; the state is conveyed by the native element, not the glyph.
  • Give every summary a clear, unique label so it makes sense out of context in a screen reader's element list.

Screen reader notes

  • NVDA + Firefox and VoiceOver + Safari announce each summary as a button with its state, e.g. "What should I bring, button, collapsed".
  • Toggling announces the new state ("expanded" / "collapsed"); the body is read when the item is open.
  • Because the +/− icon is aria-hidden, screen readers never read the glyph, only the native state.
  • Single-open mode closes siblings silently; that is expected and does not move focus, so it does not disrupt a screen reader's place.
