In-page nav

Stable

An "on this page" table of contents for long, single-page content. It lists the page's section headings and, as you scroll, highlights the one in view — orientation plus quick jumps, no page splitting.

  • css/components/nav.css
  • Needs js/wire.js (scroll-spy)
  • Since v1.0

Live demo

The static shape: a titled list with a guide line and the in-view section marked by an accent bar. The On this page rail beside this text is itself a working instance — scroll the tab and watch its highlight track the section in view.

Anatomy

A labeled nav holds a title and a list of in-page anchors. The script reads each link's #id and tracks the matching section.

PartClassRole
Rail.wire-inpagenav + data-wire-inpagenavA <nav aria-label="On this page">; the hook turns on scroll-spy.
Title.wire-inpagenav__titleThe "On this page" label.
List.wire-inpagenav__listThe anchors, behind a vertical guide line.
Link.wire-inpagenav__linkAn in-page anchor (href="#section-id"); .is-current marks the section in view.

When to use

  • Use it on long, single-page content with several headed sections — a condition overview, a guide, a long specialty page.
  • It gives orientation and quick jumps without splitting the topic into child pages.
  • Every target section needs an id; the script tracks whichever is in view.
  • For child pages rather than sections, use Side nav; for site-level destinations, the Top nav.
  • Short pages don't need one — a TOC for three sections is overhead.

Do / Don't

Do

  • Give every target heading a stable id.
  • Label the nav "On this page" and keep it to one heading level.
  • Let the script set .is-current — don't hard-code it in production.

Don't

  • List more than ~8 entries — the TOC becomes its own scroll problem.
  • Force it sticky on mobile; it's non-sticky below 48em by design.
  • Use it for cross-page navigation.

Structure

  • A vertical list behind a thin guide line; each link carries a transparent leading border that turns to --color-accent when its section is current.
  • Links are --text-sm and muted at rest, darkening on hover and on the current item.
  • The title is an uppercase --text-2xs eyebrow.

Behavior

  • data-wire-inpagenav turns on initInPageNav: an IntersectionObserver watches the linked sections and sets .is-current on the link whose section is in view.
  • It's sticky only at 48em and up (offset by --space-lg); below that it scrolls with the page so a tall TOC never eats a phone viewport.
  • The highlight is presentational — it doesn't move focus or rewrite the URL.

Snippet

<nav class="wire-inpagenav" data-wire-inpagenav aria-label="On this page">
  <p class="wire-inpagenav__title">On this page</p>
  <ul class="wire-inpagenav__list" role="list">
    <li class="wire-inpagenav__item"><a class="wire-inpagenav__link" href="#overview">Overview</a></li>
    <li class="wire-inpagenav__item"><a class="wire-inpagenav__link" href="#care-team">The care team</a></li>
  </ul>
</nav>

Landmark & labeling

  • Wrap it in <nav aria-label="On this page"> so it's a distinct landmark, separate from the primary and section navs.
  • The links are ordinary in-page anchors, so they work with or without the script.

Scroll-spy & keyboard

  • The current-section highlight is presentational only — it never moves focus or changes the URL, so it can't disrupt a keyboard or screen-reader user mid-read.
  • Enter on a link jumps to the section the normal way; give the target a heading so landing there makes sense.
  • Don't rely on the highlight alone to convey position — it's a convenience layered over real anchors.

Targets & reflow

  • Links clear a 28px target (min-block-size: 1.75rem); WCAG 2.5.8 sets a 24px minimum.
  • Going non-sticky below 48em keeps the TOC from obscuring content on small screens (WCAG 1.4.10 reflow, 2.4.11 focus not obscured).
