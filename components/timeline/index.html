Timeline

Stable

A vertical sequence of milestones, each a date, a title, and a short body. Use it for chronologies — an institution's history, a treatment journey, a project plan.

  • css/components/timeline.css
  • Since v1.0

Live demo

An ordered list with a connecting line and a node per item. Each milestone reads date, then title, then body.

Default
  1. 1953

    Founded as the Rehabilitation Institute of Chicago

    A small inpatient unit opens with a singular focus: rehabilitation as its own medical specialty.

  2. 1991

    First ranked #1 in rehabilitation

    U.S. News & World Report begins ranking; the Institute takes the top spot and holds it.

  3. 2017

    Becomes Riverside Medical Center

    A new building, a new name, and a translational model — research and care under one roof.

  4. 2026

    36th consecutive #1 ranking

    More than three decades of unbroken leadership in rehabilitation medicine.

Anatomy

PartClassRole
List.wire-timelineAn <ol> — the order is the meaning.
Item.wire-timeline__itemOne milestone; the connecting line and node are drawn in CSS.
Date.wire-timeline__dateThe milestone's date or label.
Title.wire-timeline__titleThe milestone headline.
Body.wire-timeline__bodyA short supporting line.

When to use

  • Use a timeline for a chronological sequence — institutional history, a care journey, a plan with dates.
  • Each item is a date, a short title, and a brief body; it reads top to bottom in a consistent direction.
  • For ordered steps in a process use a stepper; for unordered peers use a list.

Do / Don't

Do

  • Order items consistently (usually oldest to newest).
  • Give each a date and a short, scannable title.
  • Keep bodies to a line or two.

Don't

  • Use a timeline for unordered items.
  • Bury long paragraphs inside a milestone.
  • Mix chronological directions within one timeline.

Structure

  • Each item carries a leading border that forms the spine; a ::before node sits on the line at the item's start.
  • The first item drops its top padding and the last drops its trailing border, so the spine begins and ends at the content.
  • The node is a ring: --color-bg fill with a --color-accent border, so it reads in grayscale.

Tokens

PropertyToken
Item padding--space-lg block · --space-2xl inline-start
Spine--border-thin --color-border
Node--border-md --color-accent ring
Date--text-xs, --tracking-wider, --color-text-muted
Title--text-lg, --weight-semibold
Body--text-md, --color-text-muted, 65ch measure

Snippet

<ol class="wire-timeline" role="list">
  <li class="wire-timeline__item">
    <div class="wire-timeline__date">1953</div>
    <h3 class="wire-timeline__title">Founded as the Rehabilitation Institute</h3>
    <p class="wire-timeline__body">A small inpatient unit opens.</p>
  </li>
</ol>

Semantics

  • Use an ordered list (<ol>): the chronology is part of the meaning, and the list conveys count and order.
  • Item titles are real headings at the level that fits the page outline, so the timeline is navigable by heading.

Decorative spine

  • The connecting line and node dots are generated in CSS (a border plus a ::before), so they're decorative and never announced.
  • The date is real text, not an image, so it's read in order with each item.

Contrast

  • The node ring uses the accent token against the page background; it reads in both themes without relying on hue.
  • Date and body use muted text that still clears AA against the surface.
