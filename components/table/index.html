Table

Stable

A workhorse data table and a comparison table for plans, programs, and feature matrices. Standard HTML semantics with a grayscale treatment; both scroll horizontally on small screens via the required wrapper.

  • css/components/table.css
  • Since v1.0

Live demo

The default data table: an uppercase header row, hairline rows, and a numeric column aligned with tabular figures. The wrapper handles horizontal overflow.

When to use

  • Use a data table when rows and columns both carry meaning — stats, rosters, schedules, trials.
  • Use the comparison variant for plans, programs, or feature matrices; the featured column draws the recommended option.
  • Always wrap a table in .wire-table-wrap so it scrolls on small screens instead of breaking the layout.
  • For simple key–value facts use a definition list; for navigation rows, a linked list.

Do / Don't

Do

  • Wrap every table in .wire-table-wrap.
  • Use th[scope] and a caption.
  • Right-align numbers with .wire-table__num.

Don't

  • Use a table for page layout.
  • Omit the wrapper — the table will overflow on mobile.
  • Rely on the check / cross glyph alone without text for assistive tech.

Structure

  • The wrapper is required: it owns the border and radius and sets overflow-x: auto so wide tables scroll rather than overflow.
  • The header row sits on a subtle fill; rows are separated by hairlines and lift to the subtle fill on hover.
  • --striped shades even rows, --compact tightens vertical padding, and --comparison centers value columns and styles a featured column header.

Snippet

<div class="wire-table-wrap">
  <table class="wire-table">
    <caption class="u-visually-hidden">Annual ranking</caption>
    <thead>
      <tr><th scope="col">Year</th><th scope="col" class="wire-table__num">Patients</th></tr>
    </thead>
    <tbody>
      <tr><td>2026</td><td class="wire-table__num">52,140</td></tr>
    </tbody>
  </table>
</div>

Semantics

  • Use real table semantics: thead / tbody, th[scope="col"] for column headers and th[scope="row"] for row headers (the comparison's feature names).
  • Give every table a <caption>; use .u-visually-hidden when a visible caption isn't wanted.
  • Numeric columns use .wire-table__num so digits align and compare cleanly.

Comparison marks

  • The check (✓) and cross (—) are CSS-generated glyphs. Put visually-hidden "Yes" / "No" text inside each span so a screen reader announces the value, not a symbol.
  • The featured column is marked by an inverse fill and its position and heading — never by color alone, which the grayscale system couldn't carry anyway.

Overflow

  • The wrapper's horizontal scroll keeps a wide table from overflowing a small screen (WCAG 1.4.10 reflow); it's required, not optional.
  • A scrollable region is keyboard-focusable so it can be scrolled without a pointer.
