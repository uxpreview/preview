Citation list

Beta

A numbered reference list for bibliographies and evidence sections — each entry has the same shape: a leading number, then title, authors, and source. It's the human-readable render of the system's evidence model.

  • css/components/citation-list.css
  • Beta — promotes at a third adopter
  • Since v1.1

Live demo

A numbered list of references. Each item is a number column and a stacked body: title, authors, source meta, and optional tag chips for what the source supports.

Anatomy

PartClassRole
List.wire-citation-listAn <ol> of references.
Item.wire-citation-list__itemA two-column row: number, then body.
Number.wire-citation-list__numThe reference number, in a mono tabular column.
Title.wire-citation-list__titleThe work's title, optionally linked.
Authors.wire-citation-list__authorsAuthor or publisher.
Meta.wire-citation-list__metaPublication, date, and the canonical link.

When to use

  • Use a citation list for a bibliography or evidence section — a dense, uniform author/title/source list.
  • It's the render layer for the evidence model: in these docs it lists the standards and research the system cites.
  • The optional tag chips summarize what each source supports — handy in a design rationale sheet.
  • For a handful of inline references, footnotes or a plain list may be lighter.

Do / Don't

Do

  • Number entries in order and keep each to title, authors, and source.
  • Link the canonical URL in the meta line.
  • Mark any draft source UNVERIFIED — pending human review.

Don't

  • Present an unverified citation as established fact.
  • Mix citation styles within one list.
  • Use it for general navigation rows — that's a linked list.

Status

This component is beta. It's in use on the Evidence model and directory pages; per the manifest's promotion policy it graduates to stable once a third independent page adopts it. The class API is settled; expect no breaking changes, only wider adoption.

Structure

  • An ordered list; each item is a CSS grid with a fixed 2.5rem number column and a flexible body.
  • Items are separated by a hairline rule; the last item drops its rule and trailing padding.
  • The number is real text set in a mono, tabular column so multi-digit numbers stay aligned and survive copy-paste.

Snippet

<ol class="wire-citation-list" role="list">
  <li class="wire-citation-list__item">
    <div class="wire-citation-list__num">1</div>
    <div class="wire-citation-list__body">
      <p class="wire-citation-list__title">WCAG 2.2</p>
      <p class="wire-citation-list__authors">World Wide Web Consortium.</p>
      <p class="wire-citation-list__meta">Recommendation, 2023. <a href="https://www.w3.org/TR/WCAG22/">w3.org/TR/WCAG22</a></p>
    </div>
  </li>
</ol>

Semantics

  • Use an ordered list (<ol>): references are numbered and cited by number from the body text.
  • The number is real text, not a generated counter, so it's announced and copyable.
  • The canonical source is a real link in the meta line; link the title too when it points somewhere distinct.

Evidence honesty

  • The list renders citations.json — the single source of evidence. Never present a citation as verified fact unless it is.
  • Mark a draft source UNVERIFIED — pending human review; setting a citation verified is a human pass, never automated.
  • Tag chips label what a source supports; they're summaries, not claims of their own.

Contrast

  • Authors and meta use muted text that still clears AA against the surface in both themes.
  • Links carry the global underline-on-hover and a visible focus ring.
