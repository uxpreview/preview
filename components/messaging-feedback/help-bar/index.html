Help bar

Stable

A persistent contact bar for time-sensitive support — an icon, a short title and description, and one or two actions — that can pin to the top or bottom of the viewport.

  • css/components/help-bar.css
  • 2 variants

Live demo

An in-flow help bar: a decorative icon, a title and description, and an actions row pairing a primary action with a phone link. On a real page this is the always-available path to a person.

Help bar — in flow
?
Not sure where to start? A care navigator can point you to the right service — no cost to talk, 8 AM – 6 PM weekdays.
Request a callback 312.238.8000

Anatomy

The bar stacks an icon, a text block (title plus description), and an actions row. It is a row at tablet width and up, and a vertical stack on narrow phones so each part gets full width.

PartClassRole
Container.wire-help-bar + variantThe bar; takes an optional placement modifier.
Icon.wire-help-bar__iconDecorative mark (aria-hidden="true"); not the accessible name.
Text.wire-help-bar__textWraps the title and description.
Title.wire-help-bar__titleShort prompt — a question or offer. Never an h1.
Description.wire-help-bar__descOne line of supporting detail — hours, cost, scope.
Actions.wire-help-bar__actionsOne or two actions — buttons, with a tap-to-call number carried as a secondary button.

When to use

  • Use it for an always-available support path — a nurse line, admissions help, a patient navigator — that should stay reachable as someone reads a page.
  • Pair a single primary action with a tap-to-call number so the bar offers both a digital and a voice route to a person.
  • Reach for it on detail and task pages where someone might get stuck and need to talk to a human.

Don't use it for transient status — a dismissible system message belongs in an Alert bar or Banner, not a standing contact affordance. The help bar is for the persistent path, not the passing notice.

Variants

One bar, two placements. The default sits in the page flow; --fixed pins it to the bottom-right of the viewport. The example below shows the in-flow bar — a pinned bar is described rather than demonstrated, since it would otherwise float over this documentation.

PlacementClassBehavior
Default.wire-help-barSits in normal document flow; place it near the end of a content region.
Fixed.wire-help-bar--fixedPins to the bottom-right of the viewport on every screen; respects the iPhone home-indicator safe area. One per page.

Do / Don't

Do

  • Keep the title a short, plain-language prompt and the description to one line.
  • Pair a primary action with a tap-to-call number so people can choose how to reach a person.
  • If you pin it (--fixed), leave a clear way to reach whatever it covers.

Don't

  • Mark the bar's title up as an h1 — use .wire-help-bar__title.
  • Use it for dismissible status or alerts; that's a Banner or Alert bar.
  • Stack more than two actions, or let a pinned bar obscure the content beneath it.

Semantics

  • Wrap the bar in a complementary or region landmark with an accessible name when it stands apart from the main flow, so screen-reader users can find and skip it. The accessible name comes from the title text, not the decorative icon.
  • The icon is decoration (aria-hidden="true"); the title carries the meaning. Keep the title as text, never a heading — it isn't a level in the page outline.
  • It isn't an aria-live region. Only announce changes if the bar's content updates in place; a standing contact bar should stay silent.

Targets

  • Every action is a real link or button, sized to at least 44 px so it's easy to hit on touch and for people with limited dexterity (WCAG 2.2 SC 2.5.8, Target Size).
  • Focus stays visible on the bar's dark surface — the component re-aliases --color-focus to the inverse text color so the ring never disappears (WCAG 2.2 SC 2.4.7).
  • The phone link uses tel: so it dials directly on a phone and reads as a phone number to assistive tech.

Persistence

  • A persistent bar must not trap focus; keyboard users tab through it and out into the page in the normal order.
  • The --fixed variant pins to the viewport, so leave a way to reach anything it covers — don't let the bar obscure content or controls beneath it.
  • The fixed variant respects the device safe-area inset so it never sits on the home-indicator gesture strip.
