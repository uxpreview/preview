Skeleton

Beta

Grayscale placeholder shapes for content that is still loading. A skeleton does two things at once: it signals the page is working, not broken, and it reserves the layout so real content drops in without a jump. Use it when you know the shape of what's coming and roughly how long it will take.

  • css/components/skeleton.css
  • Atom · no JavaScript

Overview

A skeleton is an honest promise about what's coming: it mirrors the shape of the content so the wait feels shorter and the layout doesn't lurch when data lands. It earns its keep only when you know that shape and the wait is short and bounded. When you don't, a spinner is the more honest signal.

When to use

  • You know the layout that's loading — a card, a list row, a profile header.
  • The wait is short and likely to succeed, so the placeholder will be replaced soon.
  • Preventing layout shift matters — reserving the space keeps the page from jumping.

When not to use

  • The wait is unbounded or unknown — show a spinner or a progress bar instead.
  • You don't know the shape of the result — a skeleton that doesn't match is worse than none.
  • The region is empty by design, not loading — that is an empty state.

Anatomy

A single element per placeholder shape, with a modifier picking the shape; a group helper stacks several into a mock of a real block.

PartClassRole
Shape.wire-skeleton + a modifierOne placeholder block; the modifier sets its size and radius. Carries the shimmer.
Group.wire-skeleton-groupA vertical stack with consistent gaps — assemble shapes into a card or row mock.

Do / Don't

Do

Mirror the real layout: image, title, three lines.

The placeholder matches what arrives, so nothing shifts on swap.

Don't

Skeleton an open-ended wait.

If you can't say how long or what shape, a spinner is the honest signal.

States

What each state means lives in Usage › Overview.

StateTriggerWhat changes
Shimmering(default)The lighter band sweeps left-to-right on a 1.5s loop.
Reduced motionprefers-reduced-motionThe sweep stops; the static gray block stays visible.

JavaScript

None. The skeleton is CSS-only. Your loading code owns the swap: render the skeleton inside an aria-busy="true" region, then replace it with the real content and set aria-busy="false" when the data arrives.

Code

<div aria-busy="true" aria-live="polite">
  <div class="wire-skeleton wire-skeleton--image"></div>
  <div class="wire-skeleton-group u-mt-md">
    <div class="wire-skeleton wire-skeleton--title"></div>
    <div class="wire-skeleton wire-skeleton--text"></div>
    <div class="wire-skeleton wire-skeleton--text"></div>
  </div>
</div>
<!-- On load: replace the inner markup, set aria-busy="false". -->

ARIA & semantics

  • The skeleton shapes carry no text and no role — they are decorative placeholders. The meaning lives in the loading state of their container.
  • Wrap the placeholder in a container with aria-busy="true" while loading, and set it to false once the real content is in.
  • Add aria-live="polite" to that container (or the results region) so the arrival of content is announced without stealing focus.
  • Don't pepper the page with aria-label="loading" on every shape — one busy region is enough, and quieter.

WCAG 2.2 criteria mapped

SCHow Skeleton satisfies it
1.4.1 Use of ColorPlaceholders are grayscale; nothing relies on hue.
2.3.3 Animation from InteractionsThe shimmer is disabled under prefers-reduced-motion; the static block remains.
4.1.3 Status MessagesThe aria-busy / live-region pattern announces busy-to-loaded without moving focus.

Screen reader notes

  • The shapes themselves are silent — a screen reader shouldn't narrate a wall of empty boxes. The container's aria-busy conveys "loading".
  • When content replaces the skeleton inside a polite live region, the reader hears the new content rather than the placeholder.
  • Keep the placeholder structure close to the real layout so the announced result matches what sighted users see appear.
  • Pending: a live NVDA + VoiceOver pass of the busy-to-loaded swap is not yet logged — this is why Skeleton is beta.
