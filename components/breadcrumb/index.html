Breadcrumb

Stable

A trail of links showing where the current page sits in the site hierarchy, with a one-tap way back up the tree. The last item is the current page and isn't a link.

  • css/components/breadcrumb.css
  • Since v1.0

Live demo

A trail from the site root to the current page. Ancestors are links; the current page is a plain, emphasized label.

Four-level trail
  1. Home
  2. Conditions we treat
  3. Neurological
  4. Spinal cord injury

Anatomy

A labeled navigation landmark wraps an ordered list. Separators are drawn in CSS between items, never typed into the markup.

PartClassRole
Landmark.wire-breadcrumbA <nav> with aria-label="Breadcrumb".
List.wire-breadcrumb__listAn <ol> — order conveys depth.
Item.wire-breadcrumb__itemOne crumb; the separator is generated before each item after the first.
Link.wire-breadcrumb__linkA link to an ancestor page.
Current.wire-breadcrumb__currentThe current page as a non-link span with aria-current="page".

When to use

  • Use a breadcrumb on deep hierarchies — three or more levels — where users need to know where they are and step back up.
  • Skip it on flat sites and top-level pages: a one-link trail is noise.
  • It complements primary navigation, it doesn't replace it — and it's never the only way back up.
  • For a linear sequence (a multi-step form, paged results) reach for In-page nav, Pagination, or a stepper instead.

In context

A breadcrumb sits directly above the page title, below the top nav, so the trail reads as a preamble to the heading.

Above a page title
  1. Home
  2. Research
  3. Active studies

Active studies

Open trials in neurorehabilitation, accepting referrals now.

Do / Don't

Do

  • Mirror the site's hierarchy, not the user's click path.
  • Make the last crumb the current page, as a non-link.
  • Keep labels short and matched to the page titles they point to.

Don't

  • Link the current page to itself.
  • Type "/" separators into the markup — they're drawn in CSS.
  • Use a breadcrumb for steps in a flow.

Structure

  • An ordered list (<ol>): the sequence is meaningful, so position is part of the semantics.
  • Separators are generated with .wire-breadcrumb__item + .wire-breadcrumb__item::before (a / glyph) — decorative, never in the DOM text.
  • Links carry a 28px minimum tap target via min-block-size: 1.75rem, with a negative inline margin so the hit area grows without shifting the row.

Snippet

<nav class="wire-breadcrumb" aria-label="Breadcrumb">
  <ol class="wire-breadcrumb__list">
    <li class="wire-breadcrumb__item"><a class="wire-breadcrumb__link" href="/">Home</a></li>
    <li class="wire-breadcrumb__item"><a class="wire-breadcrumb__link" href="/conditions">Conditions we treat</a></li>
    <li class="wire-breadcrumb__item">
      <span class="wire-breadcrumb__current" aria-current="page">Spinal cord injury</span>
    </li>
  </ol>
</nav>

Landmark & labeling

  • Wrap the trail in <nav aria-label="Breadcrumb"> so it's exposed as its own navigation landmark, distinct from the primary nav.
  • Use an ordered list — the depth order is part of the meaning.
  • Mark the current page with aria-current="page" on a non-link <span>.

Separators

  • The / dividers are generated in CSS (::before), so they aren't selectable, focusable, or announced — exactly right for decoration.
  • Don't add separator characters to the markup; a screen reader would read each one aloud.

Keyboard & target size

  • Links sit in source order and are reachable with Tab; the current-page span is not focusable.
  • Each link clears a 28px target (min-block-size: 1.75rem); WCAG 2.5.8 sets a 24px minimum.
  • Focus shows a visible ring inherited from the global link focus style.
