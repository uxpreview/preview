Top nav

Stable

The site masthead: a sticky bar with the brand, the primary menu, and utility actions. A menu item can open a mega menu — a grouped panel for wide information architectures. Below the desktop breakpoint the menu collapses to a burger that opens the mobile drawer.

  • css/components/nav.css
  • Needs js/wire.js (mega menu)
  • Since v1.0

Live demo

The desktop masthead. The first menu item, "Care", opens a mega menu — click it to toggle the panel; Esc closes it. The primary menu is visible at wide widths; narrow the window and it gives way to a burger.

Anatomy

A sticky <header> holds the brand, the primary navigation, and utility actions. Any menu item can be promoted to a mega menu.

PartClassRole
Bar.wire-topnavSticky masthead <header>.
Inner.wire-topnav__innerFlex row: brand, menu, utility.
Brand.wire-topnav__brand · __brand-markHome link with a letter or logo mark.
Menu.wire-topnav__menu · __item · __linkPrimary nav; .is-current marks the active section.
Utility.wire-topnav__utilityTrailing actions (search, sign-in, a phone link).
Burger.wire-topnav__burgerMobile trigger carrying data-wire-drawer-open.
Mega trigger.wire-megamenu + data-wire-megamenuA menu item that opens a grouped panel.
Mega panel.wire-megamenu__panel · __grid · __col · __title · __linkThe dropdown of grouped links with optional descriptions.

Responsive behavior

  • The primary .wire-topnav__menu is hidden below 64em and shown as a horizontal row at and above it.
  • Below 64em the .wire-topnav__burger appears; it carries data-wire-drawer-open and opens the off-canvas mobile menu — the same pattern documented under Drawer.
  • The bar is position: sticky with an opaque background and a bottom border as the separator, and it respects the iOS safe-area inset.

When to use

  • Every page opens with a top nav — it's the constant masthead in the page-shell scaffold.
  • Promote an item to a mega menu only when a section fans out into many destinations that benefit from being grouped and previewed — a multi-audience care or admissions hub.
  • Keep a flat dropdown or a plain link when a section has just a handful of children; a mega menu for three links is overkill.
  • For navigation within a section, reach for Side nav; for position on a long page, In-page nav.

Do / Don't

Do

  • Label the primary nav (aria-label="Primary") and mark the current section.
  • Group mega menu links under short, scannable headings.
  • Keep the trigger a real link so it works before JS loads.

Don't

  • Stack more than a few top-level items — depth belongs in the menu, not the bar.
  • Open a mega menu on hover alone; it toggles on click for touch and keyboard.
  • Hide the only path to a section behind a mega menu with no fallback link.

The bar

  • Sticky to the top (position: sticky, z-index: var(--z-sticky)) with an opaque --color-bg and a --color-border bottom edge.
  • Row height steps up at the desktop breakpoint: min-block-size 3.5rem on mobile, 4.5rem at 48em.
  • Menu links are 44px-tall targets (min-block-size: 2.75rem) with a pill radius; .is-current and an open mega trigger both take a muted fill.

The mega menu

  • The panel is absolutely positioned just below the trigger, a bordered card with --shadow-md at z-index: var(--z-dropdown).
  • It's hidden by default; .wire-megamenu.is-open .wire-megamenu__panel reveals it.
  • The grid is two columns, widening to three at 64em; the panel's minimum width steps from 32rem to 48rem.
  • Column titles are an uppercase --text-2xs label; links carry an optional __link-desc sub-line.

Landmarks & labeling

  • The masthead is a <header>; wrap the primary navigation in <nav aria-label="Primary">.
  • When a page carries more than one nav, give each a distinct label so they're tellable apart.
  • Mark the active top-level item with .is-current and aria-current="page".

Mega menu behavior

  • The trigger stays an <a href> for a no-JS fallback; initMegamenu adds aria-haspopup="true" and toggles aria-expanded.
  • It opens on click (not hover), so touch and keyboard reach it the same way.
  • Esc closes the panel and returns focus to the trigger; a click outside closes it too.
  • Panel links follow the trigger in source order, so Tab moves straight into them once it's open.

Targets & focus

  • Top-level links clear a 44px target; mega menu links sit at a comfortable 2.25rem.
  • Focus is visible on every link and the trigger, inherited from the global focus style.
  • The skip link at the top of the page jumps past the masthead to the main content.
