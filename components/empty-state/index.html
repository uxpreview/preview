Empty state

Beta

What to show when a list, search, or section has nothing in it. A good empty state does three jobs at once: it explains why the space is blank, it reassures the user the page didn't break, and it offers a clear way forward. A blank panel does none of those.

  • Molecule · no JavaScript

Overview

An empty result is a fork in the user's experience: handled well, it tells them what happened and what to do next; handled badly, it reads as a broken page. The empty state turns a dead end into a next step — name the situation, then offer the action that resolves it.

When to use

  • A list or search with no items — by design (nothing booked yet) or by filter (nothing matches).
  • A first-run section the user hasn't populated — no saved items, no history.
  • Any place a blank region would otherwise look like a failure.

When not to use

  • While the data is still loading — that is a skeleton.
  • When something went wrong fetching it — that is an error, shown in a banner or callout with a recovery action.
  • As decoration on a page that already has content.

Anatomy

An optional grayscale visual, a title, a sentence of body, and an actions cluster — centered by default, left-aligned inline.

PartClassRole
Root.wire-emptyThe centered container; caps its width and centers in its column.
Visual.wire-empty__visualOptional grayscale shape or glyph. Decorative — keep aria-hidden="true".
Title.wire-empty__titleNames the situation in a few words. A real heading.
Body.wire-empty__bodyOne sentence: why it's empty and what to do.
Actions.wire-empty__actionsThe path forward — one or two buttons.

Types & variants

VariantSelectorUse for
Default.wire-emptyCentered blank slate for a primary region or full panel.
Inline.wire-empty--inlineLeft-aligned, tighter — inside a card, a results column, or a table replacement.
Bordered.wire-empty--borderedAdds a dashed boundary so the empty region reads as intentional, not missing.

--inline and --bordered combine; the demo above pairs --bordered with the default and uses --inline on its own.

In context

  • hospital-search.html — a no-results empty state when filters return nothing.
  • hospital-portal.html — first-run empty states on a fresh patient dashboard.
  • hospital-404.html — the error archetype borrows the empty-state shape for its dead end.

Do / Don't

Do

Explain and offer a path: "No visits yet — find a doctor."

The user knows why it's blank and what one click will fix it.

Don't

Leave it bare: "No data."

Two words with no next step reads like a bug, not a state.

Content guidelines

  • Title the situation, not the absence: "No upcoming visits" beats "Empty".
  • In the body, say why and what next in one sentence.
  • Make the primary action the most likely fix — for a filtered search, "Clear filters"; for a first run, the create action.
  • Match the tone to the cause. A filter miss is neutral; a first-run is welcoming.

States

The empty state has no interactive states of its own — it is content. The only variation is structural (default / inline / bordered), covered under Usage › Types & variants. Its action buttons carry the system's standard focus and hover states.

JavaScript

None. The empty state is pure markup and CSS. If you swap it in to replace results, do that announcing-swap in your own code (see Accessibility › Screen reader notes); the component itself ships no script.

Code

<div class="wire-empty wire-empty--bordered">
  <div class="wire-empty__visual" aria-hidden="true">∅</div>
  <h3 class="wire-empty__title">No upcoming visits</h3>
  <p class="wire-empty__body">Find a doctor to book one, or call 312.555.1000.</p>
  <div class="wire-empty__actions">
    <a class="wire-button" href="#">Find a doctor</a>
    <a class="wire-button wire-button--secondary" href="#">Browse conditions</a>
  </div>
</div>

ARIA & semantics

  • The title is a real heading (<h2>/<h3> to fit the page outline), so it appears in the heading list and names the region.
  • The visual glyph is decorative and carries aria-hidden="true" — the meaning lives in the text.
  • The actions are real links or buttons, reachable and labelled by their text.
  • When the empty state replaces live results, put it in a container with aria-live="polite" so the change is announced.

WCAG 2.2 criteria mapped

SCHow Empty state satisfies it
1.3.1 Info and RelationshipsHeading, body, and actions are real elements with a clear structure.
1.4.1 Use of ColorHierarchy comes from scale, weight, and space — no color. Grayscale throughout.
2.4.6 Headings and LabelsThe title describes the situation, not just "Empty".
4.1.3 Status MessagesWhen swapped in for results, a polite live region announces the change without moving focus.

Screen reader notes

  • Static empty states are read in source order like any content — the heading orients the user.
  • If a filter change replaces a result list with an empty state, wrap the results region in aria-live="polite" so the "no results" message is announced, not silently swapped.
  • Pending: a live NVDA + VoiceOver pass of the dynamic-swap case is not yet logged — this is why Empty state is beta.
